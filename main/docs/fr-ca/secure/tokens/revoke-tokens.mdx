Une fois émis, les jetons d’accès et d’ID ne peuvent pas être révoqués de la même façon que les témoins avec identifiant de session pour les sessions côté serveur.

Par conséquent, les jetons doivent être émis pour des périodes relativement courtes, puis [actualisés](/security/tokens/refresh-tokens) régulièrement si l’utilisateur reste actif.
