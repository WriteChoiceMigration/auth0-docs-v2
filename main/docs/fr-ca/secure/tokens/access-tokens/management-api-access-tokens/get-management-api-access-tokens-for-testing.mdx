Lorsque vous terminez la configuration dans Auth0 Dashboard, vous obtenez un jeton de test pour Management API. Vous ne devez le faire qu’une fois, sauf si vous créez un nouveau locataire. Nous vous recommandons de créer un jeton de test exclusivement pour autoriser l’accès à Management API au lieu de réutiliser un autre jeton que vous pourriez avoir.

## Prérequis

* [Créer des applications machine-machine pour les tests](/apis/create-m2m-app-test)

## Obtenir des jetons d’accès manuellement

1. Dans votre Auth0 Dashboard, naviguez jusqu’à **Applications &gt API > **[Auth0 Management API](%24%7Bmanage_url%7D/#/apis/management/explorer). Sélectionnez l’onglet **Explorateur d’API ** et localisez un jeton généré automatiquement dans la section **Jeton**.
2. Sélectionnez l’icône **Copier** à droite du jeton. Vous pouvez maintenant effectuer des appels autorisés à la [Management API](/api/management/v2) à l’aide de ce jeton.
3. , mettez à jour **l’expiration du jeton (Secondes)** et cliquez sur **Update &amp (Mettre à jour & Regenerate Token (Régénérer le jeton)**.



## Utiliser des jetons d’accès pour les tests

Vous pouvez utiliser le jeton d’accès que vous venez de créer à des fins de test pour appeler manuellement un point de terminaison via l’[Explorateur de Management API](/api/management/v2).

1. Accédez à l’[Explorateur de Management API](/api/management/v2#!) et sélectionnez le bouton **Définir le jeton API**.
2. Remplissez le champ **Token API**. Des informations supplémentaires s’affichent alors dans la fenêtre contextuelle, telles que les permissions accordées à l’application.
3. Sélectionnez **Définir le jeton API** pour enregistrer votre jeton et fermer la fenêtre contextuelle. Remarquez que le bouton Définir le jeton API s’affiche désormais sous la forme **Autorisé**.
4. Pour commencer les tests, naviguez jusqu’à un point de terminaison que vous souhaitez appeler et saisissez tous les paramètres requis ou facultatifs dans les champs de droite. Ensuite, sélectionnez **Tester le point de terminaison**.
