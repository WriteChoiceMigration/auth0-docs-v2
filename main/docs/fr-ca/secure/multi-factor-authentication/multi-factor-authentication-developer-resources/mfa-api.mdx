Auth0 fournit un flux intégré d’inscription et d’authentification multifacteur (MFA) à l’aide de la [Connexion universelle](/login/universal-login). Utilisez [MFA API](/api/authentication#multi-factor-authentication) dans les scénarios suivants si vous le souhaitez :

* [Authentifier les utilisateurs avec l’octroi du mot de passe du propriétaire des ressources](/mfa/ropg-mfa).
* Créer une interface permettant aux [utilisateurs de gérer leurs propres facteurs d’authentification](/mfa/ropg-mfa/manage-authenticator-factors-mfa-api).

Avant que vous ne puissiez utiliser les API MFA, vous devrez activer le type d’autorisation MFA pour votre application. Pour activer l’autorisation la MFA dans Auth0 Dashboard, rendez-vous à [Dashboard > Applications > Paramètres avancés > Types d’autorisation](%24%7Bmanage_url%7D/#/applications) et sélectionnez **MFA**. Consultez [Activer l’authentification multifacteur (MFA)](/mfa/enable-mfa) pour en savoir plus.

## Limites

L’API MFA est conçue pour fonctionner avec les facteurs SMS, vocaux, notifications poussées via l’application Gardien Auth0, courriel et OTP. À l’heure actuelle, elle ne prend pas en charge l’enregistrement avec Duo ou WebAuthn.
