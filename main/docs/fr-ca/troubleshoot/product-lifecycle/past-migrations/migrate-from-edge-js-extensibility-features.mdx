Depuis le 21 juin 2023, Auth0 ne prend plus en charge les fonctionnalités d’extensibilité .NET et C# Node.js.

## Flux de travail affectés

Passez en revue vos fonctionnalités d’extensibilité pour le code qui contient `require(edge)`.

Consultez les flux ou les fonctionnalités suivants qui autorisent le code d’extensibilité pour les références à `edge.js` :

* [Crochets](https://auth0.com/docs/customize/hooks)
* [Règles](https://auth0.com/docs/customize/rules)
* [Scripts de base de données personnalisés](https://auth0.com/docs/authenticate/database-connections/custom-db/create-db-connection#create-database-action-scripts)
* [Connexions sociales personnalisées](https://auth0.com/docs/authenticate/identity-providers/social-identity-providers/oauth2)



## Mises à jour recommandées

Il n’y a pas de remplacement officiel prévu pour cette fonction.

L’une des voies de migration consiste à remplacer la logique `edge` par un service distinct au sein de votre propre infrastructure. Après avoir créé le service dans votre infrastructure, vous pouvez utiliser une fonction d’extensibilité pour effectuer une requête HTTP afin de déclencher l’opération requise.

Par exemple, dans un script de connexion à une base de données personnalisée utilisant `edge`, supprimez la fonction qui appelle `edge` avec un client HTTP générique pour capturer la réponse.

Une autre voie de migration consiste à réécrire la fonction d’extensibilité en JavaScript au lieu de .NET/C#.

Pour plus d’informations sur la création d’Actions ou la migration vers des Actions, [consultez la documentation Auth0](https://auth0.com/docs/customize/actions).
