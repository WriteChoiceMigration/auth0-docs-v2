## Problèmes liés à l’utilisateur

### Si vous n’avez pas votre appareil mobile ou si votre appareil mobile est éteint

Si vous avez perdu votre appareil, vous pouvez terminer l’authentification en utilisant le code de récupération fourni lors de votre inscription.

1. Enter your email and password to log in, and click the **Use the recovery code** link.
2. Enter your recovery code.

Si vous n’avez plus votre code de récupération, vous ne pourrez pas vous connecter. Contactez votre administrateur système pour obtenir de l’aide afin d’accéder à votre compte.

### Si vous avez oublié votre mot de passe

Si vous avez oublié votre mot de passe, cliquez sur le lien **Mot de passe oublié ou perdu?** situé sous les champs courriel et mot de passe. Saisissez ensuite votre adresse courriel pour recevoir un courriel contenant un lien qui vous permettra de réinitialiser votre mot de passe.

### Si votre transaction expire

When logging in via <Tooltip tip="Multi-factor authentication (MFA): User authentication process that uses a factor in addition to username and password such as a code via SMS." cta="View Glossary" href="/docs/glossary?term=MFA">MFA</Tooltip>, there is a five-minute maximum between providing your first and second factors. You can see how much time has elapsed since you logged in using the first factor by checking the timestamp on the messages provided.

Si plus de cinq minutes se sont écoulées, vous devrez vous connecter à nouveau et obtenir un nouveau code ou une nouvelle notification.

Si vous demandez des SMS, assurez-vous que vous ne dépassez pas les limites tarifaires.

### Si vous devez retirer ou supprimer la MFA d’un utilisateur de votre locataire

If you need to remove, delete, or reset MFA for a user, you should [reset MFA](/docs/secure/multi-factor-authentication/reset-user-mfa).

## Problèmes liés aux SMS

### Vous n’avez pas reçu de message via SMS

Si vous n’avez pas reçu votre code à six chiffres par SMS, vérifiez que le numéro de téléphone que vous avez fourni est correct. Si c’est le cas, assurez-vous que vous êtes dans une zone couverte par le réseau cellulaire.

Si vous ne recevez toujours pas les messages, vérifiez auprès de votre fournisseur de services que les messages ne sont pas bloqués.

### Limites anti-attaques des SMS

Si vous tentez d’envoyer plus de dix SMS à votre appareil en l’espace d’une heure, vous verrez apparaître un message d’erreur concernant une exception à la limite anti-attaques.

Lorsque vous dépassez votre limite de messages, vous devez attendre au moins une heure après la demande de votre premier message avant d’en demander un autre. Vous recevrez une tentative additionnelle après le passage de chaque heure additionnelle.

## Codes rejetés

Si le code à 6 chiffres figurant dans le Guardian ou dans l’application Google Authenticator est rejeté pour la connexion (souvent avec le message `Incorrect Code`), vérifiez d’abord que vous sélectionnez la bonne application dans votre application d’authentification. Si vous avez vérifié que vous avez sélectionné la bonne application, assurez-vous que les réglages de l’horloge de votre appareil mobile sont corrects. Les mots de passe à usage unique sont générés en utilisant le temps universel coordonné (UTC), l’heure de votre appareil doit donc être correcte pour que votre code fonctionne.

Pour vérifier les réglages de votre horloge :

* **Appareils Android** - Allez à **Paramètres** > **Date et heure**. Assurez-vous que la case en regard de **Automatique** est cochée. Pour la désactiver, allez dans **Paramètres > Date et heure**. Touchez la case à côté de **Automatique** pour la décocher.
* **Appareils iOS** - Allez à **Paramètres** > **Général** > **Date et heure**. Activez **Régler automatiquement**. Si ce paramètre était déjà activé, vous pouvez le désactiver pendant un moment, puis le réactiver.
