En cas de fonctionnalités d’Auth0 devenues obsolètes, il peut y avoir des erreurs ou des avis dans les journaux des locataires qui s’affichent pour indiquer que vos applications utilisent des fonctionnalités obsolètes. Ce guide vous aidera à rechercher dans vos journaux les messages liés à l’obsolescence, ainsi que des explications sur les causes potentielles et les solutions pour des éléments particuliers.

Il existe deux façons différentes de rechercher des messages d’avertissement indiquant l’utilisation de fonctionnalités obsolètes : Dashboard ou Management API. Notez que dans les deux cas, la [période de rétention des journaux](/logs/log-data-retention) est régie par le niveau d’abonnement de votre compte.

## Utilisation du Dashboard

Si votre application utilise une fonctionnalité obsolète, un **avis d’obsolescence** apparaîtra dans la section **Journaux** du Dashboard. Les avis d’obsolescence ne sont affichés qu’une fois par heure (la première fois que l’erreur se produit au cours de cette heure) et non pour chaque transaction d’authentification impliquant la fonctionnalité obsolète.

1. Allez à [Dashboard > Surveillance > Journaux](https://manage.auth0.com/#/logs) et recherchez les messages relatifs à l’obsolescence en saisissant `type:depnote` dans la boîte de recherche. Une liste de messages d’avertissement liés à l’obsolescence et provenant de vos journaux s’affichera, s’il en existe. Le champ **Description** fournit des informations sur la fonctionnalité obsolète utilisée.
2. Cliquez sur le lien dans la colonne **Type** pour chaque élément afin d’afficher des informations supplémentaires telles que l’ID client, qui identifie l’application client utilisant la fonctionnalité obsolète.
3. Cliquez sur les entrées de journal et sélectionnez l’onglet **Données contextuelles** pour afficher les détails de l’élément.

## Utilisation de Management API

Vous pouvez également utiliser Management API pour rechercher ces messages dans les journaux en recherchant `type:"depnote"`.

1. Allez à [Management API](/api/management/v2/). Si ce n’est déjà fait, [obtenez un jeton d’API](/security/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production).
2. Sur la gauche, allez à [Journaux > Rechercher les événements de journaux](/api/management/v2/#!/Logs/get_logs), puis faites défiler vers le bas jusqu’à **Paramètres**.
3. Dans le champ **q**, saisissez : `type:"depnote"`.
4. Cliquez sur la touche **Essayer**. Le champ **ID client** dans les résultats indiquera quelle application de votre locataire utilise la fonctionnalité obsolète.
