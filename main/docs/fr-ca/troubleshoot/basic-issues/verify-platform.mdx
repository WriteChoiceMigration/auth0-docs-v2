Assurez-vous d’utiliser les versions les plus récentes de votre pile technologique. Si vous utilisez une ancienne version d'un langage de programmation ou d'une bibliothèque, pensez à la mettre à jour.

* Check the [Auth0 support matrix](/docs/troubleshoot/customer-support/product-support-matrix) to see if the language or library versions you're using is supported.
* S'il y a un problème dans un environnement (comme le développement) mais pas dans un autre (comme la production), comparez les versions de la pile technologique dans les deux environnements.
* Si ce n’est pas le cas, mettez à jour votre infrastructure technologique et votre navigateur avec les dernières versions et testez à nouveau avant de créer une demande de soutien.

## Vérifiez l’étendue du problème

Testing on different browsers, platforms, locations, and users can help narrow down the source of a problem when it occurs. Perform the following tests and review the [logs in your Auth0 dashboard](https://manage.auth0.com/#/logs) after each test for more information.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

See [the Monitoring page](/docs/deploy-monitor/monitor) for more details on each of the log events that can help you troubleshoot issues.

</Callout>

### Testez avec différents navigateurs

* Le problème se produit-il avec tous les navigateurs ou seulement avec certains d’entre eux?
* Testez dans une nouvelle session de navigation incognito ou privée (sans état antérieur).
* Passez à la version la plus récente du navigateur et refaites le test.
* Désactivez les plugiciels du navigateur et refaites un test pour voir si un plugiciel contribue au problème.

### Testez avec différents utilisateurs

* Le problème se produit-il pour tous les utilisateurs ou seulement pour certains d’entre eux?
* Si c’est le cas, existe-t-il un dénominateur commun? Par exemple, l’emplacement, la connexion ou le rôle?

### Testez avec différentes connexions (si possible)

* Le problème se produit-il avec toutes les connexions ou avec une seule?

### Testez avec différentes applications clientes (si vous en avez plus d’une)

* Le problème se produit-il avec tous les clients ou avec un seul?

### Testez dans différents environnements

* Essayez les environnements de développement, d’essai et de production de votre (vos) application(s).
* Essayez avec différents locataires Auth0 pour voir s’il y a des différences.

## Vérifiez si le problème est constant ou intermittent

* Si le problème est constant, notez les étapes à suivre pour reproduire le problème de manière fiable.
* Si le problème est intermittent, notez tout point en commun qui pourrait en être la cause.
* S’agit-il d’un problème lié à une nouvelle configuration, ou bien le système fonctionnait-il auparavant et est-il à présent défectueux?

  + Si le problème vient de se produire, quelles modifications ont été apportées?
  + L’annulation des modifications résout-elle le problème?

## Problèmes affectant un seul ou quelques utilisateurs

* [Check the user's profile](/docs/troubleshoot/authentication-issues/check-user-profiles), browser, or device for any issues.
* Vérifiez si cela se produit dans tous les navigateurs pour les utilisateurs affectés (indiquant un problème de données) ou uniquement dans certains types de navigateurs (indiquant un problème spécifique au navigateur).
* Vérifiez si JavaScript et les témoins sont activés dans le navigateur.
* Vérifiez que la touche de verrouillage des majuscules est désactivée.
* Si l'utilisateur utilise un appareil mobile, vérifiez s'il existe un logiciel susceptible d'avoir une incidence sur l'authentification et/ou l'autorisation (par exemple, s'il n'exécute pas un logiciel requis).
* Check to see if the user can access some of the application's key URLs, such as the <Tooltip tip="Identity Provider (IdP): Service that stores and manages digital identities." cta="View Glossary" href="/docs/glossary?term=identity+provider">identity provider</Tooltip>'s <Tooltip tip="Single Sign-On (SSO): Service that, after a user logs into one applicaton, automatically logs that user in to other applications." cta="View Glossary" href="/docs/glossary?term=Single+Sign-on">Single Sign-on</Tooltip> (SSO) URL (indicating a network connectivity issue).

## Problèmes survenant après la mise en service (fonctionnait avant, puis a cessé de fonctionner)

* Vérifiez si des changements ont été apportés récemment à votre application ou aux API que vous appelez.
* Vérifiez s’il y a eu des changements récents dans le réseau (équilibreur de charge, pare-feu, changements dans la configuration du proxy).
* Vérifiez si des changements ont été apportés récemment à l’infrastructure (par exemple, les identifiants des serveurs LDAP ou des serveurs de base de données).
* Vérifiez s’il existe des correctifs ou des mises à jour des applications, de l’infrastructure matérielle ou des infrastructures technologiques.
* Vérifiez que tous vos serveurs utilisent le protocole NTP et qu’ils sont synchronisés avec l’heure exacte.
* Vérifiez si l’un de vos certificats a expiré.
* Vérifiez auprès des propriétaires des fournisseurs d’identité distants si quelque chose a changé.

  + Vérifiez la connectivité réseau avec les fournisseurs d’identité distants.
* Check notifications in <Tooltip tip="Auth0 Dashboard: Auth0's main product to configure your services." cta="View Glossary" href="/docs/glossary?term=Auth0+dashboard">Auth0 dashboard</Tooltip>/migrations - any changes you overlooked?
* Vérifiez le journal des modifications d’Auth0 : y a-t-il des changements de configuration récents liés à votre problème ?
* Vérifiez si un composant de l’infrastructure technologique a été mis à jour ou corrigé récemment.
