Une transaction d’authentification comporte généralement plusieurs parties. Auth0 fournit des méthodes afin que vous testiez des parties individuelles de la transaction et déterminiez les causes des problèmes.

La plupart des connexions des fournisseurs d’identité ont un bouton **TRY** pour vérifier leur fonctionnement. En cas d’échec, vous pouvez déboguer la connexion sans faire intervenir le reste de l’application. Si la connexion fonctionne, vous pouvez commencer à déboguer l’application.

1. Utilisez le bouton **ESSAYER** sur une connexion pour ne tester que la connexion.
2. Si le test avec le bouton **ESSAYER** échoue, vous saurez que l’application n’est pas en cause, mais plutôt la configuration de la connexion ou le fournisseur de la connexion.
3. Vérifiez la réponse au test avec le bouton **ESSAYER** pour voir si la réponse contient un message d’erreur ou d’autres informations utiles.
4. Essayez de vous connecter au même service via un autre chemin.
5. Si le même problème survient, vous saurez que le compte en est la cause d’une façon ou d’une autre.
6. Si un test avec le bouton **ESSAYER** a échoué avec une connexion de base de données personnalisée, le problème se situe généralement au niveau des scripts de base de données personnalisée. Insérer des déclarations console.log à l’intérieur de ces scripts et regarder le résultat dans le journal de la console peut contribuer à résoudre le problème.

## Vérifier les dépendances de service externes

1. Si votre authentification utilise des services externes, comme des fournisseurs d’identité sociale, et qu’elle s’arrête soudainement de fonctionner, [vérifiez le statut des services externes](/monitor-auth0/check-external-services-status).
2. Si une connexion ne fonctionne pas, même avec le bouton **ESSAYER**, vérifiez la connexion.
