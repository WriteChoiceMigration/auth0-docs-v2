Si vous rencontrez des problèmes avec [](/extensions) une extension, nous vous recommandons de commencer le processus de dépannage en suivant les deux étapes suivantes :

1. Upgrade the extension, if an upgrade is possible.
2. Reinstall the extension.

To learn more about different extensions Auth0 offers, read [Auth0 Extension](/docs/customize/extensions).

## Réinstaller l’extension

L’une des premières choses que vous pouvez faire lorsque vous rencontrez des problèmes avec une extension est de la réinstaller :

1. Go to [Auth0 Dashboard > Extensions](https://manage.auth0.com/#/extensions).
2. On the **Installed Extensions** tab, check if there is an upgrade available for the specific extension.
3. Click the **Update to...** option.

## Migrer vers la version 12 de Node.js

Nous vous recommandons de modifier l’exécution de l’extensibilité de votre locataire de la v. 8 de Node.js à la v. 12 de Node.js. Avant de mettre à jour, cependant, consultez le [guide de migration](/product-lifecycle/deprecations-and-migrations/migrate-to-nodejs-12) pour obtenir tous les détails sur ce qui sera affecté par ce changement.

1. Go to [Auth0 Dashboard > Extensions](https://manage.auth0.com/#/extensions).
2. On the **Installed Extensions** tab, delete the extension.
3. Log out of the Auth0 Dashboard.
4. Log back in to the Auth0 Dashboard.
5. Reinstall and reconfigure the extension.

## Vous pouvez modifier la version d’exécution de Node.js en accédant à la section **Extensibilité** dans [Auth0 Dashboard > Paramètres > Avancés]().

If you are still experiencing issues with your extension, and you need to [contact Auth0 support](https://support.auth0.com), please be sure to **include errors logs and/or** [**Actions Real-time Logs**](/docs/customize/actions/actions-real-time-logs) **with your support ticket**. Errors and logs can help Auth0 support troubleshoot your issue faster.
