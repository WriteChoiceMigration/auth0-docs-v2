Use [Auth0 Support Center](https://support.auth0.com) to create tickets for questions or issues. All tenant member roles can access Support Center.

Si vous êtes un client existant du nuage privé, vous devrez créer un compte Auth0 basé sur le nuage pour vous connecter au Centre d’assistance. Ce compte peut également être utilisé à des fins de développement et de test, sans frais supplémentaires. Veuillez communiquer avec votre responsable de compte technique ou votre directeur des ventes pour associer votre compte basé sur le nuage à votre abonnement existant au Nuage privé.

## Ouvrir un nouveau ticket

1. Go to [Support Center](https://support.auth0.com) and select **Open Ticket**.

   <Callout icon="file-lines" color="#0EA5E9" iconType="regular">

   If you don't see this button, then you do not have access to open support tickets (only paying and trial customers have access to open support tickets). In this case, select **Ask our Community** to use our [Community](https://community.auth0.com/) resources.

   </Callout>
2. Select the tenant you need help with from the **Affected Tenant** dropdown menu.
3. Under **What can we help you with?**, select the type of issue that best fits your case.
4. If prompted, choose the appropriate [severity level](#severity-levels) (not available for trial tenant customers).
5. Provide a clear summary of the request or issue in the **Request Summary** field.
6. Provide as much detail as possible about the issue or request in the **Description** box. When writing in this box, you can style your text with [Markdown](https://guides.github.com/features/mastering-markdown) and preview what you have written by clicking **Preview**. When writing your description, please try to include the following information when it applies:

   1. What were you trying to do?
   2. What did you expect to happen?
   3. Where did things go wrong?
   4. Were there any error messages or screenshots showing the problem? If so, please include them.
   5. Has this worked before, or are you trying a new configuration?
   6. Does the problem occur for all users or just a few?
   7. What troubleshooting steps have you tried?
7. When you have completed all the fields, select **SUBMIT**.

## Mise à jour d’un ticket existant

1. Go to the [Support Center](https://support.auth0.com) page and select **Home**.
2. Select the ticket that you want to update by selecting its **Request Summary**.
3. Enter any additional details into the text box and then select **REPLY**.

Si vous êtes le demandeur du ticket et que le ticket est assigné à un agent mais n’est pas résolu ou fermé, vous avez la possibilité de changer le statut de votre ticket comme **Résolu** en cochant la case **Soumettre comme résolu** à côté du bouton **RÉPONDRE**.

## Rouvrir un ticket fermé

Si votre ticket a été fermé mais que vous souhaitez continuer à traiter le problème avec Auth0, créez un nouveau ticket dans le Centre d’assistance. Assurez-vous d’inclure le numéro de référence du ou des tickets originaux.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

When a case is closed, Auth0 permanently deletes all file attachments for security purposes.

</Callout>

## Gérer les tickets d’abonnement

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

To manage subscription tickets, you must have the [Elevated Support Access role](/docs/get-started/manage-dashboard-access/feature-access-by-role) assigned to you.

If you are a Tenant Administrator, you can assign the Elevated Support Access role to yourself to see all tickets opened by all users on your tenants.

</Callout>

Pour gérer les tickets d'abonnement :

1. Go to [Auth0 Support Center](https://support.auth0.com)
2. Select **Tickets** from the navigation menu.
3. Switch to the **Subscription Tickets** view.
