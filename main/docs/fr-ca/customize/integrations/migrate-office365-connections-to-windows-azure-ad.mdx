Office365 est obsolète. Vous devriez migrer vos connexions Office365 vers les connexions Windows Azure AD.

Depuis le commencement, nous nous chargeons de l’authentification des utilisateurs avec Office365. Office365 a toujours utilisé Windows Azure AD en coulisses, mais il n’y avait pas une bonne interface utilisateur pour créer une « application » dans Windows Azure AD. C’est pourquoi vous avez dû la créer dans le tableau de bord du vendeur. Microsoft souhaite maintenant que vous utilisiez Windows Azure AD; vous pouvez désormais facilement créer un répertoire associé à votre compte Office365 et à l’application.

## Comment migrer vers Azure

1. Create a Windows Azure AD subscription (free)
2. Create a Directory (that will be associated with your Office365 account) and an application. To learn more, read [Connect Your App to Microsoft Azure Active Directory](/docs/authenticate/identity-providers/enterprise-identity-providers/azure-active-directory/v2).

Si vous utilisiez le `user_id` dans votre application, remarquez qu’il passera de `office365|....some-guid....` à `waad|...email....`.
