Auth0 vous permet de configurer votre propre fournisseur de messagerie SMTP afin de mieux gérer, surveiller et dépanner vos communications par courriel. L’infrastructure de messagerie intégrée d’Auth0 ne doit être utilisée que pour les courriels de test. Vous ne pouvez configurer qu’un seul fournisseur de messagerie qui sera utilisé pour tous les courriels.

Auth0 prend actuellement en charge les fournisseurs suivants :

* [Mandrill](/brand-and-customize/email/smtp-email-providers/configure-mandrill-as-external-smtp-email-provider)
* [Amazon SES](/brand-and-customize/email/smtp-email-providers/configure-amazon-ses-as-external-smtp-email-provider)
* [Azure Communication Services](/configure-azure-comm-service-as-smtp-email-provider)
* [Microsoft 365](/configure-365-exchange-as-smtp-email-provider)
* [SendGrid](/brand-and-customize/email/smtp-email-providers/configure-sendgrid-as-external-smtp-email-provider)
* [SparkPost](/brand-and-customize/email/smtp-email-providers/configure-sparkpost-as-external-smtp-email-provider)
* [Mailgun](/brand-and-customize/email/smtp-email-providers/configure-mailgun-as-external-smtp-email-provider)

Vous pouvez également configurer un [fournisseur de messagerie tiers](/brand-and-customize/email/configure-custom-external-smtp-email-provider) comme Gmail ou Yahoo.

Vous configurez les fournisseurs de messagerie externes dans [Auth0 Dashboard > Image de marque > Fournisseur de courriel](%24%7Bmanage_url%7D/#/templates/provider).
