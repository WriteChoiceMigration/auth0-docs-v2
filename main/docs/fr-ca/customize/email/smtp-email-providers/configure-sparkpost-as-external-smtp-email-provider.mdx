Auth0 vous permet de configurer votre propre fournisseur de messagerie SMTP afin que vous puissiez gérer, surveiller et dépanner plus en détail vos communications par courriel. L’infrastructure de messagerie intégrée d’Auth0 ne doit être utilisée que pour les courriels de test.

## Prérequis

* Pour que les courriels puissent être envoyés de Auth0 à votre SMTP, des ports doivent être ouverts et il faut autoriser les connexions entrantes provenant d’adresses IP spécifiques. Pour mettre à jour la liste des adresses IP, accédez à [Dashboard > Image de marque > Fournisseur de messagerie électronique](%24%7Bmanage_url%7D/#/branding/email_provider) Voir [Ajouter des adresses IP à AllowList](/security/data-security/allowlist) pour plus de détails.
* Vous devez disposer d’un compte [](https://www.mandrill.com/signup/)[](https://mailgun.com)[SparkPost](https://www.sparkpost.com/).

## Configurer SparkPost

1. Connectez-vous à votre compte [SparkPost](https://www.sparkpost.com/).
2. Dans SparkPost, allez à [Domaines d’envoi](https://app.sparkpost.com/account/sending-domains) et ajoutez votre domaine personnalisé. SparkPost n’autorise l’envoi de courriels qu’à partir de domaines vérifiés.
3. Dans SparkPost, allez à [Clés d’API de comptes](https://app.sparkpost.com/account/credentials) et cliquez sur **Nouvelle clé API**. Enregistrez la valeur de cette clé et assurez-vous qu’elle dispose de l’accès `Transmissions: Read/Write`. Copiez la valeur de la clé.
4. Accédez à Auth0 [Dashboard > Image de marque > Fournisseur de messagerie électronique](%24%7Bmanage_url%7D/#/branding/email_provider).
5. Activez la bascule **Utiliser mon propre fournisseur de messagerie**.
6. Cliquez sur le logo **SparkPost**.
7. Fournissez une adresse courriel dans le champ **De**, saisissez dans le champ **Clé API** la clé API SparkPost que vous avez copiée précédemment, puis sélectionnez votre **Région**. Si vous changez la région de US à EU pour un compte déjà configuré, vous devez remplacer la **clé API** dans Auth0 par une clé API EU Sparkpost.
8. Cliquez sur **Enregistrer**.

## Courriel de test

Vous pouvez maintenant envoyer un courriel de test en utilisant le bouton **Envoyer un courriel de test**. Si vous avez tout configuré correctement, vous recevrez un courriel de confirmation. Si vous ne recevez pas de courriel après quelques minutes, consultez les [journaux d’Auth0](%24%7Bmanage_url%7D/#/logs) pour voir si des erreurs sont répertoriées.

If you haven’t registered a custom domain with SparkPost, you can send a maximum of five test emails from the `sparkpostbox.com` domain. For more details, see SparkPost’s [Transmissions: The Sandox Domain](https://developers.sparkpost.com/api/transmissions.html#header-the-sandbox-domain).

La page [Événements de message](https://app.sparkpost.com/reports/message-events) SparkPost affichera des informations de livraison pour tous les courriels envoyés à vos utilisateurs.
