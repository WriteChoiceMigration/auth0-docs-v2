Le Dashboard à authentification unique (SSO) est une application Web conçue pour résoudre un problème familier à de nombreuses personnes. Les organisations de toutes tailles maintiennent une variété d’applications différentes pour gérer diverses fonctions commerciales telles que la comptabilité, les ressources humaines, le développement, le soutien, etc. Se souvenir des noms d’utilisateur, des mots de passe et des URL de connexion pour toutes vos applications peut être fastidieux. Avec cette extension, vous pouvez simplifier l’expérience d’authentification en activant la connexion à SSO pour vos utilisateurs sur plusieurs applications d’entreprise. Elle vous permet de créer une liste de tous les services infonuagiques pour lesquels un utilisateur peut s’authentifier avec des connexions à SSO.



Le Dashboard à authentification unique prend en charge deux types d’utilisateurs :

* Les **utilisateurs** qui peuvent se connecter au tableau de bord pour sélectionner une application à laquelle se connecter avec la SSO.
* Les **administrateurs** qui peuvent configurer des applications visibles par les utilisateurs.

Pour configurer cette extension, procédez comme suit :

1. [Créez une application Dashboard à authentification unique](/extensions/single-sign-on-dashboard-extension/create-sso-dashboard-application)
2. [Installez l’extension Dashboard à authentification unique](/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension)
3. [Ajoutez les applications au Dashboard à authentification unique](/extensions/single-sign-on-dashboard-extension/add-applications-to-the-sso-dashboard)
4. [Mettez à jour les applications dans le Dashboard à authentification unique](/extensions/single-sign-on-dashboard-extension/update-applications-on-the-sso-dashboard)

Vous pouvez également [afficher cette extension de Dashboard à authentification unique sur GitHub](https://github.com/auth0-extensions/auth0-sso-dashboard-extension).
