Utilisez [](/extensions/single-sign-on-dashboard-extension)l’extension du Dashboard SSO pour gérer la connexion à signature unique de vos utilisateurs sur plusieurs applications d’entreprise.

Pour en apprendre davantage, consultez l’[extension du Dashboard à authentification unique (SSO)](/extensions/single-sign-on-dashboard-extension) ou [affichez cette extension sur GitHub](https://github.com/auth0-extensions/auth0-sso-dashboard-extension).

## Prérequis

* [Créez l’application Dashboard SSO](/extensions/single-sign-on-dashboard-extension/create-sso-dashboard-application) dans Auth0.
* [Installez l‘extension du Dashboard SSO](/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension).
* [Ajoutez les applications au Dashboard SSO](/extensions/single-sign-on-dashboard-extension/add-applications-to-the-sso-dashboard).

## Mise à jour de l’application

1. Allez à [Auth0 Dashboard > Extensions](%24%7Bmanage_url%7D/#/extensions) et choisissez la vue **Extensions installées**.
2. Sélectionnez **Tableau de bord SSO**.
Si vous êtes un administrateur, vous pouvez également vous connecter au Tableau de bord SSO en utilisant `https://${account.tenant}.<REGION>8.webtask.io/auth0-sso-dashboard/admins/login`.
3. Sélectionnez **Paramètres** à partir du menu déroulant se trouvant sous votre nom de locataire.

* Pour spécifier si les utilisateurs peuvent voir l’application une fois activée, sélectionnez **Publier** ou **Ne pas publier**.
* Pour mettre les paramètres de l’application à jour, sélectionnez l’icône de la roue d’engrenage.
* Pour supprimer une application, sélectionnez son **X**. Une fenêtre contextuelle apparaîtra pour confirmer la suppression.
