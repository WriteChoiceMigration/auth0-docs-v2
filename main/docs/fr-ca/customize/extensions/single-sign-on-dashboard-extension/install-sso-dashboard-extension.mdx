Utilisez l’extension Dashboard à authentification unique pour gérer la connexion SSO de vos utilisateurs sur plusieurs applications d’entreprise. Pour en savoir plus, [Consultez cette extension sur GitHub](https://github.com/auth0-extensions/auth0-sso-dashboard-extension).

## Installer l’extension

Vous pouvez installer l’extension Dashboard à authentification unique par le biais de l’Auth0 Dashboard :

1. Accédez à [Auth0 Dashboard > Extensions](%24%7Bmanage_url%7D/#/extensions) et sélectionnez **Dashboard à authentification unique** dans la liste des extensions fournies. La fenêtre **Installer l’extension** s’ouvre :
2. Définir les variables de configuration suivantes :
3. Sélectionnez **Installer**

Si vous accédez à [Auth0 Dashboard > Applications > Applications](%24%7Bmanage_url%7D/#/applications), vous remarquerez qu’une application nommée `auth0-sso-dashboard` a été créée automatiquement. Cette application est utilisée par l’extension pour autoriser l’accès à Management API, et vous ne devez pas la modifier.

## Utilisation de l’extension

1. Accédez à [Auth0 Dashboard > Extensions](%24%7Bmanage_url%7D/#/extensions), et sélectionnez la vue **Extensions installées**.
2. Sélectionnez **Dashboard à authentification unique**. Lorsque vous sélectionnez votre extension installée, il vous sera demandé de lui accorder les autorisations requises. Une fois que vous avez accepté, vous serez dirigé vers votre **Dashboard à authentification unique** personnalisé, qui affichera le **Titre** que vous avez fourni en haut de la page. Si vous avez fourni un fichier CSS personnalisé, ce style sera également appliqué.
3. Pour ouvrir une session dans le Dashboard à authentification unique :

1. Pour **Administrateurs**, utilisez : 
`https://${account.tenant}.YOUR_REGION_KEY.webtask.io/auth0-sso-dashboard/admins/login`
2. Pour **Utilisateurs**, utilisez : 
`https://${account.tenant}.YOUR_REGION_KEY.webtask.io/auth0-sso-dashboard/login
`
