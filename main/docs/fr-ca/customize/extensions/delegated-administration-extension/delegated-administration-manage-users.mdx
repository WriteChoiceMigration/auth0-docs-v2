Deux affichages sont proposés dans l’application exposée par l’extension d’administration déléguée : **Utilisateurs** et **Journaux**. Dans l’aperçu **Utilisateurs**, vous pouvez afficher et modifier les utilisateurs associés à votre compte Auth0.

Par défaut, tous les utilisateurs sont affichés, mais vous pouvez filtrer la liste affichée en configurant un hook de filtre

Pour plus d’informations sur le crochet de filtre, consultez [Administration Déléguée : Hook de filtre](/extensions/delegated-administration-extension/delegated-administration-filter-hook).

## Actions des utilisateurs dans le tableau de bord de l’administration déléguée

Pour limiter le nombre d’options possibles pour quelqu’un ayant accès au Dashboard exposé par l’extension d’administration déléguée, configurez un hook d’accès.

Lisez davantage sur les hooks d’accès dans [Administration déléguée : Hook d’accès](/extensions/delegated-administration-extension/delegated-administration-access-hook).

L’option **Modifier le profil** n’est proposée que si vous avez configuré des champs personnalisés.

Vous remarquerez la nouvelle option **Réinitialiser le mot de passe** proposée par l’extension. Cette option enverra un courriel à l’utilisateur lui permettant de choisir un nouveau mot de passe. Pour ce faire, sélectionnez un utilisateur, puis choisissez **Actions** > **Réinitialiser le mot de passe**.

Cela enverra un courriel à l’utilisateur contenant un lien pour modifier le mot de passe.

Si votre profil indique que vous avez le rôle `Delegated Admin - Administrator` , l’aperçu **Journaux** vous permet de voir une liste d’authentifications effectuées par vos utilisateurs (cette page n’est visible que pour les utilisateurs ayant le rôle `Delegated Admin - Administrator`). Le contenu de cet aperçu est un sous-ensemble des données affichées dans [Auth0 Dashboard > Surveillance > Journaux](%24%7Bmanage_url%7D/#/logs). Le tableau de bord des journaux affiche également des données sur les actions administratives effectuées dans Auth0 Dashboard.

## Créer des utilisateurs

Vous pouvez créer un nouvel utilisateur en sélectionnant le bouton **+ Créer un utilisateur** dans l’aperçu **Utilisateurs**. Vous devez préciser le courriel et le mot de passe. Selon votre rôle, vous ne pourrez peut-être pas définir le **Service** auquel appartient le nouvel utilisateur.

Par exemple, les utilisateurs ayant le rôle `Delegated Admin - Administrator` peuvent voir le champ **Service** et sélectionner n’importe laquelle de ses valeurs.

En revanche, Kelly, qui joue le rôle `Delegated Admin - User` et fait partie du Service des finances, ne peut pas voir le champ **Service** . L’utilisateur ainsi créé sera automatiquement affecté au Service des finances.
