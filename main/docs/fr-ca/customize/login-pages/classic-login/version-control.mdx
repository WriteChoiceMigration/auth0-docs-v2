Vous pouvez utiliser un logiciel de contrôle de version pour gérer le code source de vos pages de connexion classiques.  L’utilisation du contrôle de version pour votre contenu peut être particulièrement bénéfique si vous utilisez un flux CI/CD pour la configuration de votre locataire et que vous avez personnalisé le HTML d’une ou plusieurs pages de connexion classiques.

Pour mettre en œuvre le contrôle de version, utilisez l’intégration Auth0 Marketplace avec la méthode de votre choix :

* [(Pipelines GitLab)](https://marketplace.auth0.com/integrations/gitlab-pipeline)
* [Actions GitHub](https://marketplace.auth0.com/integrations/github-actions)
* [Pipelines Bitbucket](https://marketplace.auth0.com/integrations/bitbucket-pipeline)
* [Pipelines Microsoft Azure](https://marketplace.auth0.com/integrations/azure-pipeline)

Le processus général de déploiement comprend les étapes suivantes :

1. Créez un dossier dans votre référentiel de contrôle de version avec le nom approprié (`pages`).
2. Créez une page HTML (`login.html`, `password_reset.html`, `guardian_multifactor.html`, ou `error_page.html`) dans ce dossier.
3. Créez un fichier JSON portant le même nom que votre page HTML pour chaque page hébergée que vous souhaitez intégrer au contrôle de version. Pour activer la page, le fichier JSON doit contenir les éléments suivants :

`  {
    "enabled": true
  }`

Par exemple : 
`your-repo/pages/error_page.html
your-repo/pages/error_page.json`
