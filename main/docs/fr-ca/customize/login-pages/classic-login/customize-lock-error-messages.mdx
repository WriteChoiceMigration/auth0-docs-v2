## Erreurs personnalisées dans les Règles

Si vous renvoyez des codes d’erreur personnalisés à partir d’une [règle](/rules) ou d’un script de base de données personnalisé, vous pouvez gérer les erreurs personnalisées :

* Dans l’URL de redirection de votre application, en lisant les paramètres de chaîne de requête `error` et `error_mesage`.
* En redirigeant l’utilisateur vers vos pages hébergées avec un message d’erreur personnalisé et en affichant le message avec un [message flash](/libraries/lock/lock-api-reference).
