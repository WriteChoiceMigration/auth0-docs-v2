Par défaut, **Lock.Android** affiche tout le texte en anglais. Si vous souhaitez afficher le texte dans une autre langue, vous pouvez fournir un fichier `strings.xml` et définir les valeurs à utiliser pour les différents éléments de texte que Lock pourrait afficher.

Android asks the device what locale was configured by the user and tries to fetch the list of localized texts for that language. For this to work, the developer needs to add each translation file into the app by using a special folder naming convention as per Android standards. More information can be found in the [Android developer docs](https://developer.android.com/training/basics/supporting-devices/languages).

Certaines des valeurs par défaut fournies par Lock englobent :

```xml lines
<resources>
  <string name="com_auth0_lock_social_error_authentication">Error parsing Authentication data</string>
  <string name="com_auth0_lock_social_error_title">There was an error during authentication</string>
  ...
</resources>
```






En fournissant votre propre fichier `strings.xml`, ces valeurs peuvent être ajustées comme suit :

```xml lines
<resources>
  <string name="com_auth0_lock_social_error_authentication">There was an authentication error!</string>
  <string name="com_auth0_lock_social_error_title">Social login error!!</string>
  ...
</resources>
```






## Valeurs de chaîne Lock.

For a full list of the names used by Lock, see the [default strings.xml file](https://github.com/auth0/Lock.Android/blob/master/lib/src/main/res/values/strings.xml) in the Lock.Android repository.
