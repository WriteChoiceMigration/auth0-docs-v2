The Actions Template gallery offers a variety of starter templates for creating an Action. These templates provide sample code covering various triggers and use cases, such as <Tooltip tip="Multi-factor authentication (MFA): User authentication process that uses a factor in addition to username and password such as a code via SMS." cta="View Glossary" href="/docs/glossary?term=Multi-factor+Authentication">Multi-factor Authentication</Tooltip>, Access Control, and Profile Enrichment.

Les modèles fournissent un code de départ que vous pouvez modifier et utiliser comme bloc de construction dans le cadre d’un déclencheur. Une fois que vous avez sélectionné un modèle pour votre cas d’utilisation spécifique, vous pouvez personnaliser le code pour répondre aux besoins individuels de votre application.

For a step-by-step guide on using Actions Templates, see [Write Your First Action](/docs/customize/actions/write-your-first-action).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

The Actions templates are also available at: https://github.com/auth0/opensource-marketplace.

</Callout>

## Meilleures pratiques

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

Action Templates are designed for educational purposes only.

These templates consist of incomplete code snippets, and should not be used in a production environment under any circumstances. Developers are responsible for securely developing and completing the Action Template before deploying to a production environment.

</Callout>

* Assurez-vous que les modèles sont adaptés à votre environnement spécifique. Le code étant incomplet, vous devez personnaliser et compléter les exemples avant de les importer dans un environnement de production.
* Ne déployez pas de modèle sans l’avoir modifié et complété au préalable.
* Les modèles d’Actions contiennent souvent des balises dans les commentaires, qui ne doivent pas être modifiées ou supprimées.
* Utilisez toujours la dernière version d’un package `npm`.
* Les modèles d’Actions ciblent par défaut les dernières versions de Node.js. Après avoir créé un modèle, vous pouvez le modifier pour prendre en charge d’autres versions de Node.js, si nécessaire.

Please review all [Actions Coding Guidelines](/docs/customize/actions/action-coding-guidelines) as they apply here as well.
