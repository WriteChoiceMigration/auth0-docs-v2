In general, the Deploy CLI resource configuration files closely match the payload schemas of the [Auth0 Management API](https://auth0.com/docs/api/management/v2), but there are some notable nuances to be aware of.

## Autorisations des clients

The Deploy CLI's own client grant is intentionally not exported nor configurable by itself. This is done to prevent breaking changes, otherwise the tool could potentially revoke access or otherwise crash in the midst of an import. In a multi-tenant, multi-environment context, it is expect that new tenants will have a designated client already established for the Deploy CLI, as mentioned in [Getting Started](/docs/deploy-monitor/deploy-cli-tool).

## Invites

Les invites textuelles multilingues personnalisées suivent une hiérarchie particulière. Sous la propriété de ressource des invites au niveau racine se trouve une propriété propriétaire `customText` qui est utilisée pour regrouper les traductions de texte personnalisées avec d’autres paramètres d’invites. Sous `customText` se trouve le code de langue à deux caractères. Le troisième est l’ID de l’invite, suivi de l’ID de l’écran, puis de l’ID du texte.

### Hiérarchie

```yaml lines
prompts:
  customText:
    <LANGUAGE>: # two character language code
      <PROMPT_ID>: # prompt ID
        <SCREEN_ID>: # prompt screen ID
          <TEXT_ID>: 'Some text'
```






### Exemple

```yaml lines
prompts:
  identifier_first: true
  universal_login_experience: classic
  customText:
    en:
      login:
        login:
          description: Login description in english
          buttonText: Button text
      mfa:
        mfa-detect-browser-capabilities:
          pickAuthenticatorText: 'Try another method'
          reloadButtonText: 'Reload'
          noJSErrorTitle: 'JavaScript Required'
        mfa-login-options:
          pageTitle: 'Log in to ${clientName}'
          authenticatorNamesSMS: 'SMS'
```






## Bases de données

When managing database connections, the values of `options.customScripts` point to specific javascript files relative to the path of the output folder. Otherwise, the payload closely matches that of the [Auth0 Management API](https://auth0.com/docs/api/management/v2#!/Connections/post_connections).

### Lorsque la connexion universelle est remplacée par un code HTML personnalisé, les contenus relatifs aux erreurs, à la connexion, à l’authentification multifacteur (MFA) et à la réinitialisation du mot de passe sont organisés dans des pages HTML spécifiques.

Exemple YAML

```text lines
./databases/
    /Username-Password-Authentication
        /change_password.js   
        /create.js   
        /delete.js   
        /get_user.js   
        /login.js   
        /verify.js   
./tenant.yaml
```






 Structure du dossier en mode YAML :

```yaml lines
databases:
  - name: Username-Password-Authentication
    # ...
    options:
      # ...
      customScripts:
        change_password: ./databases/Username-Password-Authentication/change_password.js
        create: ./databases/Username-Password-Authentication/create.js
        delete: ./databases/Username-Password-Authentication/delete.js
        get_user: ./databases/Username-Password-Authentication/get_user.js
        login: ./databases/Username-Password-Authentication/login.js
        verify: ./databases/Username-Password-Authentication/verify.js
```






### Contenu de `tenant.yaml` :

Exemple de répertoire

```text lines
./database-connections/
    ./Username-Password-Authentication/
        ./change_password.js
        ./create.js
        ./database.json
        ./delete.js
        ./get_user.js
        ./login.js
        ./verify.js
```






Structure du dossier en mode répertoire :

```json lines
{
  "options": {
    "customScripts": {
      "change_password": "./change_password.js",
      "create": "./create.js",
      "delete": "./delete.js",
      "get_user": "./get_user.js",
      "login": "./login.js",
      "verify": "./verify.js"
    }
  }
}
```






## Contenu de `database.json` :

### Connexion universelle d’Auth0

When overriding the <Tooltip tip="Universal Login: Your application redirects to Universal Login, hosted on Auth0's Authorization Server, to verify a user's identity." cta="View Glossary" href="/docs/glossary?term=Universal+Login">Universal Login</Tooltip> with custom HTML, the error, login, <Tooltip tip="Universal Login: Your application redirects to Universal Login, hosted on Auth0's Authorization Server, to verify a user's identity." cta="View Glossary" href="/docs/glossary?term=multi-factor+authentication">multi-factor authentication</Tooltip>, and password reset contents are organized in specific HTML pages.

#### Lorsque la connexion universelle est remplacée par un code HTML personnalisé, les contenus relatifs aux erreurs, à la connexion, à l’authentification multifacteur (MFA) et à la réinitialisation du mot de passe sont organisés dans des pages HTML spécifiques.

Exemple YAML

```text lines
./pages/
    /error_page.html
    /guardian_multifactor.html
    /login.html
    /password_reset.html
./tenant.yaml
```






Structure du dossier en mode YAML :

```yaml lines
pages:
  - name: error_page
    html: ./pages/error_page.html
    show_log_link: false
    url: https://mycompany.org/error
  - name: guardian_multifactor
    enabled: true
    html: ./pages/guardian_multifactor.html
  - name: login
    enabled: false
    html: ./pages/login.html
  - name: password_reset
    enabled: true
    html: ./pages/password_reset.html
```






#### Contenu de `tenant.yaml` :

Exemple de répertoire

```text lines
./pages/
    ./error_page.html
    ./error_page.json
    ./guardian_multifactor.html
    ./guardian_multifactor.json
    ./login.html
    ./login.json
    ./password_reset.html
    ./password_reset.json
```






Structure du dossier en mode répertoire :

```json lines
{
  "name": "login",
  "enabled": false,
  "html": "./login.html"
}
```






Contenu de `login.json` :

```json lines
{
  "html": "./error_page.html",
  "show_log_link": false,
  "url": "https://mycompany.org/error",
  "name": "error_page"
}
```






Contenu de `error_page.json` :

```json lines
{
  "enabled": true,
  "html": "./guardian_multifactor.html",
  "name": "guardian_multifactor"
}
```






Contenu de `guardian_multifactor.json` :

```json lines
{
  "enabled": true,
  "html": "./password_reset.html",
  "name": "password_reset"
}
```
