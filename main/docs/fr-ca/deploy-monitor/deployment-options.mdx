Auth0 propose à la fois des options de déploiement sur les nuages public et privé, ainsi que des outils pour vous aider dans votre déploiement d’entreprise.

La plateforme d’identité Auth0 peut être déployée de la manière suivante :

<table class="table"><thead>
<tr>
<th><strong>Option</strong></th>
<th><strong>Description</strong></th>
</tr>
</thead>
<tbody>
<tr>
<td><strong>Public Cloud</strong></td>
<td>A multi-tenant cloud service running on Auth0's cloud with standard authentication, personalization, user management, and security features</td>
</tr>
<tr>
<td><a href="/docs/deploy-monitor/deploy-private-cloud/private-cloud-on-aws"><strong>Private Cloud on AWS</strong></a></td>
<td>A dedicated, managed cloud service running on AWS providing isolation, higher performance, dev instances, Geo-HA add-ons and more</td>
</tr>
<tr>
<td><a href="/docs/deploy-monitor/deploy-private-cloud/private-cloud-on-azure"><strong>Private Cloud on Azure</strong></a></td>
<td>A dedicated cloud service running on Azure providing isolation, higher performance, dev instances, Geo-HA add-ons and more</td>
</tr>
</tbody>
</table>

Les offres **Private Cloud (Nuage privé)** sont des services gérés que vous pouvez utiliser dans les cas suivants :

* Vous ne pouvez pas utiliser un service de nuage public multilocataire dans votre organisation.
* Exiger un niveau garanti de requêtes par seconde (RPS)
