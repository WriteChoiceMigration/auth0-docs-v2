The Deploy CLI is not the only tool available for managing your Auth0 tenant configuration, there is also an [officially supported Terraform Provider](https://github.com/auth0/terraform-provider-auth0). [Terraform](https://terraform.io/) is a third-party tool for representing your cloud resources’ configurations as code. It has an established plug-in framework that supports a wide array of cloud providers, including Auth0.

Les fournisseurs de l’outil Deploy CLI et de Terraform sont là pour vous aider à gérer les configurations de votre locataire Auth0, mais chaque solution présente ses propres avantages et inconvénients.

Vous pourriez considérer le fournisseur Terraform Auth0 si :

* vos flux de travail de développement exploitent déjà Terraform;
* vos besoins en matière de gestion des locataires sont granulaires ou ne concernent que quelques ressources spécifiques.

Vous pourriez éviter le fournisseur Terraform Auth0 si :

* Votre flux de travail de développement n’utilise pas Terraform, ce qui nécessite une configuration supplémentaire dès le départ;
* Vos flux de travail de développement concernent principalement la gestion de de locataires en masse;
* votre locataire dispose de nombreuses ressources existantes, ce qui peut nécessiter d’importants efforts pour « importer ».
