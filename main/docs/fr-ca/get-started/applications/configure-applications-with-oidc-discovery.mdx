Les documents [OpenID Connect (OIDC) Discovery (Découverte OpenID Connect) ](https://openid.net/specs/openid-connect-discovery-1_0-final.html#RFC5785) contiennent des métadonnées sur le fournisseur d’identité (IdP). L’ajout de la découverte à votre trousse SDK pour pointer votre application vers le point de terminaison `./wellknown` pour consommer des informations sur votre IdP pourrait aider à configurer votre intégration avec l’IdP.

L’intégration de la découverte OIDC dans votre trousse SDK permet :

* Points de terminaison exposés du idP
* Les [Demandes et permissions soutenues par l’OIDC](/apis/scopes/openid-connect-scopes) standard (cela exclut les [demandes personnalisées](/security/tokens/json-web-tokens/create-custom-claims) et les permissions définies dans votre locataire)
* Fonctionnalités prises en charge par l’IdP

Vous pouvez configurer des applications avec les documents de découverte OpenID Connect (OIDC) trouvés ici : `https://${account.namespace}/.well-known/oauth-authorization-server`.

## Algorithme RSA pour les JWT

Le logiciel médiateur OIDC ne prend pas en charge les JWT signés avec des clés symétriques. Assurez-vous de configurer votre application pour utiliser l’algorithme RSA à l’aide de clés publiques/privées.

1. Accédez à [Dashboard > Settings Paramètres](%24%7Bmanage_url%7D/#/applications/%7BYOUR_AUTH0_CLIENT_ID%7D/settings).
2. Faites défiler vers le**Paramètres avancés**.
3. Sous l’onglet **OAuth **, définissez `RS256` comme **Algorithme de signature Demandes de jetons Web JSON (JWT)** et cliquez sur **Save (Enregistrer)**.

Avec ce paramètre, Auth0 émettra des JWT signés avec votre clé de connexion privée. Votre application les vérifiera avec votre clé de connexion publique.
