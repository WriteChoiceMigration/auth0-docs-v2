Auth0 permet aux clients disposant d’un abonnement Enterprise d’associer plusieurs locataires à un seul abonnement Auth0 (ces locataires liés peuvent également être appelés « locataires enfants »).

Cette fonctionnalité peut être utile dans les situations suivantes :

* Séparer les locataires de développement et de production tout en conservant le même accès aux fonctionnalités sur les locataires de développement que sur les locataires de production.
* Posséder plus d’un locataire de production dans le cadre d’un seul abonnement Enterprise.

## Association de locataires

L’abonnement Enterprise permet aux utilisateurs d’associer automatiquement les locataires à un abonnement Auth0 existant en sélectionnant l’option « Custom Agreement (Accord personnalisé) » dans « Create Under (Créer sous) » lors de la création de nouveaux locataires.

<Frame>![](/images/cdy7uua7fh8z/7ggqdv5yw011z0TsdlaHOO/7707d826dd585e074bd0d8e25b30d9d6/publiccloudtenants.png)</Frame>

Si vous avez besoin d’associer des locataires précédemment créés qui ne font pas partie de votre abonnement Enterprise, contactez votre responsable de compte technique désigné.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

You can also link accounts if you have a self-service tenant. To learn more, read [Tenant Management](/docs/get-started/auth0-teams/tenant-management).

</Callout>

## Consolidation des usages

L’utilisation des locataires associés est prise en compte dans les limites de l’abonnement Enterprise et est agrégée dans les rapports d’utilisation et de quota applicables.
