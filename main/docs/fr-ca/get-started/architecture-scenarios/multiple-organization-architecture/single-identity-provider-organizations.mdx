Dans ce scénario, chaque organisation exploitant la fonctionnalité [Auth0 Organizations](/organizations/organizations-overview) dispose exactement d’une connexion Auth0[](/organizations/organizations-overview). Cette fonctionnalité permet à chaque application de sélectionner l’organization correspondante configurée sur le locataire Auth0. Cette organization authentifie directement les informations d’identification de l’utilisateur ou redirige la demande d’authentification vers l’IdP approprié pour traitement.

Dans ce scénario, nous continuerons à utiliser l’exemple [Travel0 Corporate Booking](/architecture-scenarios/multiple-organization-architecture). Dans cet exemple, l’application de réservation d’entreprise correspond à votre application et Travel0 correspond à votre entreprise.



Peu importe la méthode utilisée pour traiter les informations d’identification, qu’il s’agisse d’une connexion à une base de données (comme avec Hoekstra & Associates), d’une validation via une connexion d’entreprise (comme avec MetaHexa Bank) ou d’une connexion sociale, votre application fonctionne de manière identique dans tous les cas. Tirer parti de la fonctionnalité [Organizations](/organizations/organizations-overview) vous permet de configurer et de contrôler l’expérience.
