Vous pouvez ajouter des autorisations à une API à l’aide d’Auth0 Dashboard ou de Management API.

## Utiliser le Dashboard

1. Rendez-vous à [Dashboard > Applications > API](%24%7Bmanage_url%7D/#/apis) et cliquez sur le nom de l’API à afficher.
2. Allez dans l’onglet **Autorisations** et saisissez un nom de permission et une description pour la permission que vous souhaitez ajouter. Veillez à ne pas utiliser de noms de permission réservés (voir la section Noms réservés).
3. Cliquez sur **Ajouter**. N’oubliez pas que les applications individuelles peuvent avoir besoin d’autorisatoins et/ou de permissions mises à jour pour interagir correctement avec l’API.

## Utiliser Management API



Effectuez un appel `PATCH` au [point de terminaison Update Resource Server](https://auth0.com/docs/api/management/v2/resource-servers/patch-resource-servers-by-id). Veillez à remplacer les valeurs fictives `API_ID`, `MGMT_API_ACCESS_TOKEN`, `PERMISSION_NAME` et `PERMISSION_DESC` par votre identifiant d’API, votre jeton d’accès à Management API, le(s) nom(s) de la permission et la(les) description(s) de la permission, respectivement. Veillez à ne pas utiliser de noms de permission réservés (voir la section Noms réservés).







## Noms réservés

Les noms de permission suivants sont réservés et ne peuvent pas être définis en tant que permissions API personnalisées :

* address
* created_at
* email
* email_verified
* family_name
* given_name
* identities
* name
* nickname
* offline_access
* openid
* phone
* image
* profile
