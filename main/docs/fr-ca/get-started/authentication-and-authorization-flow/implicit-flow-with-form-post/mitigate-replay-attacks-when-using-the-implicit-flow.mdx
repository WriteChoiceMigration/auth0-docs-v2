Pour réduire les attaques par réinsertion en utilisant le [Flux Implicite avec Form Post](/login/flows/implicit-flow-with-form-post), un nombre aléatoire doit être envoyé lors [des demandes d’authentification, comme exigé par OpenID Connect (OIDC)](https://openid.net/specs/openid-connect-core-1_0.html#ImplicitAuthRequest).

Le nombre aléatoire est créé par l’application, envoyé par un paramètre de chaîne de requête en tant que `nonce` dans la demande d’authentification, et inclus dans la réponse de jeton d’ID depuis Auth0. Cela permet aux applications de corréler la réponse de jeton d’ID depuis Auth0 avec la demande d’authentification initiale.

Pour en savoir plus sur l’emplacement du nombre aléatoire, voir [Ajouter une connexion en utilisant le flux Implicite avec Form Post](/login/authentication/add-login-using-the-implicit-flow-with-form-post).



## Créer un nombre aléatoire de manière cryptographique

Une des façons de créer un nonce aléatoire de manière cryptographique consiste à utiliser un outil comme [Nano ID](https://github.com/ai/nanoid) ou équivalent. Cela vous oblige toutefois à associer l’outil avec votre code JavaScript. Si cela n’est pas possible, vous pouvez profiter du fait que les [navigateurs récents](http://caniuse.com/#feat=cryptography) peuvent utiliser[ Web Crypto API](https://www.w3.org/TR/WebCryptoAPI/) pour créer des chaînes aléatoires sécurisées utilisables comme nombres aléatoires.

## Valider un jeton d’ID

Lorsque Auth0 répond avec un [jeton d’ID](/security/tokens/id-tokens), ce jeton doit être [validé et décodé](/security/tokens/id-tokens/validate-id-tokens) comme d’habitude.
Sa demande `nonce` doit contenir la même valeur exacte qui a été envoyée dans la demande.
Sinon, l’authentification sera rejetée par l’application.
