Le type d’authentification que vous souhaitez utiliser dépend de la version de Connexion universelle que vous avez configurée dans votre application. Si vous utilisez la Connexion universelle, vous pouvez configurer [l’authentification sans mot de passe à l’aide de WebAuthN avec les données biométriques de votre appareil](/login/universal-login/configure-universal-login-with-passwordless/webauthn-device-biometrics) ou utiliser les connexions sans mot de passe tel que décrit ci-dessous.

Vous pouvez configurer l’authentification sans mot de passe par courriel ou SMS en utilisant les méthodes suivantes :

1. Connexion universelle + paramètres de connexion sans mot de passe
2. Connexion classique + Lock
3. Connexion classique + IU personnalisée + Auth0.js

Pour en savoir plus à propos de ces implémentations, consultez [Configurer l’authentification sans mot de passe par courriel ou SMS](/login/universal-login/configure-universal-login-with-passwordless/email-or-sms).

### Erreurs

Si vous utilisez une connexion intégrée, nous vous recommandons d’utiliser des domaines personnalisés. Pour en savoir plus, veuillez consulter [Domaines personnalisés](/custom-domains).

## Considérations

* Vous ne pouvez pas créer d’utilisateurs sans mot de passe à partir du Auth0 Dashboard. Créez-les directement sur la [Management API](https://auth0.com/docs/api/management/v2#!/Users/post_users)
* Les jetons d’actualisation ne peuvent pas être récupérés lors de l’utilisation d’un lien magique sans mot de passe envoyé par courriel. Cela n’est possible qu’avec un mot de passe à usage unique.
