Si votre URL de rappel est valide, lorsqu’une erreur d’autorisation se produit, le serveur d’autorisations renvoie les paramètres d’erreur et d’état appropriés à votre URL de rappel. Si l’URL de rappel n’est pas valide, votre application affichera la page d’erreur générique Auth0 par défaut. Vous pouvez également personnaliser les pages d’erreur. Pour en savoir plus, consultez les [Pages d’erreur personnalisées](/brand-and-customize/custom-error-pages).

Pour connaître le nom par défaut de la page d’erreur générique, voir [Utiliser le contrôle de version pour gérer vos pages de connexion universelles](/brand-and-customize/version-control).

## Affichage

La page d’erreur générique affiche les informations suivantes si vous avez configuré les champs associés dans vos [Tenant Settings (Paramètres du locataire)](%24%7Bmanage_url%7D/#/tenant/).



## Paramètres

Le serveur d’autorisations renvoie les paramètres ajoutés à l’URL sous la forme d’une chaîne de requête.



Les paramètres présentés varient en fonction du type d’erreur et sont spécifiques à la demande. Par exemple, si la demande ne contient pas `client_id` et qu’elle aboutit à une erreur, la page d’erreur ne renverra pas d’ID client.
