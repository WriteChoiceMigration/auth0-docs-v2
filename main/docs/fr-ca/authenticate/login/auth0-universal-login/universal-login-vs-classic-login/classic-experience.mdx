La connexion classique est une expérience de connexion hébergée par Auth0 qui s’appuie sur JavaScript pour la personnalisation. La mise en œuvre d’une connexion classique est moins complexe que l’intégration du processus d’authentification directement dans votre application, et elle peut aider à prévenir les dangers de l’authentification cross-origin.

La Connexion classique est basée sur les bibliothèques Auth0 JavaScript ([Lock.js](/libraries/lock), [auth0.js](/libraries/auth0js), Widget MFA, et Réinitialisation du mot de passe). Lors de la personnalisation des pages de connexion classique dans Auth0 Dashboard, les modèles par défaut utilisent les mêmes bibliothèques JavaScript pour créer une transition plus naturelle entre l’interface utilisateur par défaut et une interface personnalisée.

Après avoir choisi un modèle par défaut pour vos pages de connexion classique, vous pouvez le modifier pour répondre à vos besoins. Vous pouvez également personnaliser une variété d’éléments comportementaux et d’apparence du gadget logiciel Lock. Les modèles Auth0.js offrent une flexibilité supplémentaire car vous pouvez créer une interface utilisateur personnalisée et la modifier pour qu’elle corresponde au style de votre application.



## Implémentation de la Connexion classique

Pour savoir comment configurer votre application pour utiliser la connexion classique, lisez nos Démarrages rapides. Choisissez l’approche qui correspond le mieux à vos technologies, et les exemples de démarrage rapide vous guideront à travers la mise en œuvre.
