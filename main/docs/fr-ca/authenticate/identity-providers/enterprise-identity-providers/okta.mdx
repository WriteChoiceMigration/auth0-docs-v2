La connexion Okta Workforce Enterprise est une intégration officiellement supportée, simplifiée, et est la méthode privilégiée de mise en place d’Okta en tant que Fournisseur d’identité (IdP) pour Auth0.



L’intégration offre aux consommateurs la possibilité de gérer les accès des employés par l’application par le biais du Workforce Identity Cloud d’Okta.

De plus, si vous utilisez Auth0 pour la gestion de l’identité client et Okta pour la gestion interne de l’identité de la main d’oeuvre, cette intégration est un moyen efficace de gérer vos espaces d’identité.

Si la connexion Okta Workforce Enterprise ne prend pas en charge votre cas d’utilisation, vous pouvez [configurer Okta en tant qu’IdP SAML](/saml-configuration/configure-auth0-saml-service-provider/configure-okta-as-saml-identity-provider) ou [configurer un serveur d’autorisation personnalisé](https://developer.okta.com/docs/concepts/auth-servers/#custom-authorization-server) pour répondre à vos besoins.

## Créer une intégration d’application OIDC Okta

Pour savoir comment créer une intégration d’application OIDC Okta, veuillez consulter l’article [Créer une intégration d’application OIDC dans Okta Help Center](https://help.okta.com/en-us/Content/Topics/Apps/Apps_App_Integration_Wizard_OIDC.htm).

Utilisez les configurations suivantes lorsque vous configurez votre intégration d’application OIDC Okta :

1. Sélectionnez **OIDC** comme méthode de connexion dans le champ **Sign-in method**.
2. Sélectionnez **Application Web** comme **Type d’application** et définissez les paramètres suivants :
3. Enregistrez le **ID client** et le **Secret client** générés par Okta pour l’intégration de votre application.

## Ajouter un utilisateur test pour l’intégration d’application Okta

Créez un utilisateur test pour votre Répertoire Okta afin de tester l’intégration d’application :

1. Dans votre Tableau de bord Administrateur Okta, naviguez vers **Répertoire > Personnes**.
2. Sélectionnez **Ajouter personne**.
3. Saisissez les détails utilisateur, y compris un mot de passe.
4. Sauvegardez l’utilisateur test.
5. Dans le **Répertoire**, sélectionnez le nouvel utilisateur
6. Accédez à l’onglet **Applications** de l’utilisateur et choisissez **Attribuer des applications**.
7. Sélectionnez le nom d’application que vous avez créé précédemment.

## Créez une connexion Okta Workforce Enterprise dans Auth0

Assurez-vous d’avoir le **ID client** et le **Secret client** de votre intégration d’application OIDC Okta disponibles :

1. Accédez à [Auth0 Dashboard > Authentification > Entreprise](%24%7Bmanage_url%7D/#/connections/enterprise), localisez **Okta Workforce**, puis sélectionnez le bouton **+**.
2. Saisissez les détails de votre connexion et sélectionnez **Créer : **
3. Dans l’affichage **Provisioning (Fourniture)**, configurez la façon dont les profils utilisateurs sont créés et mis à jour dans Auth0.

## Configurer la révocation globale des jetons

Ce type de connexion prend en charge un point de terminaison de révocation globale des jetons, qui permet à un fournisseur d’identité conforme de révoquer les sessions utilisateur Auth0, de révoquer les jetons d’actualisation et de déclencher la déconnexion du canal d’appui pour les applications utilisant un canal d’appui sécurisé.

Cette fonctionnalité peut être utilisée avec Universal Logout dans Okta Workforce Identity Cloud.

Pour plus d’informations et d’instructions de configuration, consultez[Universal Logout](/universal-logout).
