Auth0 met à disposition les API suivantes pour que les développeurs puissent les utiliser dans leurs applications.

## Authentication API

Authentication API  expose les fonctionnalités d’identité pour Auth0 et les protocoles d’identité pris en charge (y compris OpenID Connect, OAuth et SAML).

En règle générale, vous devriez utiliser cette API via l’une des trousses SDK Auth0, comme [Auth0.js](/libraries/auth0js), ou une bibliothèque telle que [Lock](/libraries/lock). Toutefois, si vous construisez votre interface d’authentification manuellement, vous devrez appeler directement Authentication API.

Exemples de tâches :

* Obtenir des [jetons](/security/tokens) lors de l’authentification
* Demander le profil d’un utilisateur en utilisant un [jeton d’accès](/security/tokens/access-tokens)
* Échanger des [jetons d’actualisation](/security/tokens/refresh-tokens) contre de nouveaux jetons d’accès
* Demander un défi-réponse pour l’[authentification multifacteur (MFA)](/mfa)

### API

Découvrez et explorez les requêtes et réponses pour les points de terminaison de Auth0 Authentication API dans votre navigateur avec [Authentication API](/auth-api).

## Management API

Management API vous permet de gérer votre compte Auth0 de manière programmable : vous pouvez ainsi automatiser la configuration de votre environnement. La plupart des tâches que vous pouvez effectuer dans Auth0 Management Dashboard peuvent également être effectuées de manière programmée en utilisant cette API.

Exemples de tâches :

* Enregistrer vos applications et API avec Auth0
* Configurez des [connexions](/connections) avec lesquelles vos utilisateurs pourront s’authentifier
* [Manage users (Gérer les utilisateurs)](/users)
* [Associer des comptes d’utilisateurs](/users/user-account-linking/link-user-accounts)

### Pagination

Pour les locataires de Public Cloud (Nuage public), les points de terminaison de Management API renvoient un maximum de 50 résultats par réponse. Si le nombre de résultats dépasse la limite pouvant être renvoyée dans une seule réponse, utilisez les paramètres `page` et `per_page` dans votre requête pour récupérer des résultats supplémentaires.

Veuillez consulter [Migrer vers les requêtes paginées des points de terminaison de Management API v2](/product-lifecycle/deprecations-and-migrations/migrate-to-paginated-queries) pour obtenir des informations sur la mise à jour de vos applications.

### La version 1 de Management API a été déconseillée

La version 1 de Management API est déconseillée et ne devrait pas être utilisée pour de nouveaux projets.

La version 1 de Management API a atteint sa fin de vie le **13 juillet 2020**. Il se peut que vous ayez été invité(e) à prendre des mesures avant cette date pour garantir la continuité du service. Des notifications ont été envoyées aux clients qui devaient effectuer cette migration.
