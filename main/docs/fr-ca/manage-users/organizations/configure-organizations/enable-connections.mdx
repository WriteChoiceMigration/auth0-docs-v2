Vous pouvez activer des connexions spécifiques pour chaque [organisation](/organizations/organizations-overview) afin de fournir différentes options de connexion aux utilisateurs. Après avoir activé une connexion, elle est ajoutée à l’invite de connexion de l’organisation et les utilisateurs peuvent s’authentifier via cette connexion pour accéder à vos applications.

Pour activer une connexion pour une organisation, la connexion **doit** déjà exister dans votre locataire. Les connexions prises en charge comprennent les [connexions de base de données](/connections/database), [connexions via un réseau social](/connections/identity-providers-social) et [connexions d’entreprise](/connections/identity-providers-enterprise).
