Utilisez le point de terminaison [Unlink a User Account (Dissocier un compte utilisateur)](/api/management/v2#!/Users/delete_user_identity_by_user_id) de Management API Auth0 ou la bibliothèque Auth0.js pour dissocier une identité du compte utilisateur cible, le transformant ainsi en un compte utilisateur distinct.

Le résultat du processus de dissociation est le suivant :

* Le compte secondaire est supprimé du tableau des identités du compte principal.
* Un nouveau compte utilisateur secondaire est créé.
* Le compte secondaire ne contiendra aucune métadonnée.

Si votre objectif est de supprimer complètement l’identité secondaire, vous devez d’abord dissocier les comptes, puis [supprimer le compte secondaire nouvellement créé](/users/delete-users).

Selon l’endroit d’où vous appelez le point de terminaison, utilisez l’une de ces deux permissions :

* `update:current_user_identities` à partir de [client-side code](/users/user-account-linking/user-initiated-account-linking-client-side-implementation)
* `update:users` à partir de [server-side code](/users/user-account-linking/suggested-account-linking-server-side-implementation)

## Utilisez le jeton d’accès avec la permission `update:users`

Si vous devez dissocier deux ou plusieurs comptes utilisateur, appelez Management API [Dissocier un point de terminaison de compte utilisateur](/api/v2#!/Users/delete_user_identity_by_user_id) en utilisant un [Jeton d’accès à Management API](/api/v2/tokens) avec la permission `update:users`.



## Dissocier les comptes du code côté serveur

1. Mettre à jour l’utilisateur en session avec le nouveau tableau d’identités (chacun représentant un compte utilisateur distinct).



1. Appelez [Dissocier un point de terminaison de compte utilisateur](/api/v2#!/Users/delete_user_identity_by_user_id) de Management API v2 en utilisant [Jeton d’accès à Management API](/api/v2/tokens) avec la permission `update:users`.
