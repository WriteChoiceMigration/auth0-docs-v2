All users associated with a single Auth0 tenant are shared between the tenant’s applications (and therefore have access to the applications).

* Pour séparer les utilisateurs et limiter leur accès, il est recommandé de [créer un locataire supplémentaire](/get-started/create-tenants/create-multiple-tenants) avec les applications auxquelles vous souhaitez que des groupes d’utilisateurs aient accès.

* Pour restreindre l’accès des utilisateurs à certaines applications, vous pouvez utiliser des [règles](/rules).

* Au sein d’une règle, les variables `context.clientName` et `context.clientID` sont disponibles pour vérifier quelle application l’utilisateur utilise pour se connecter.
* Veuillez [consulter cette règle pour un exemple](https://github.com/auth0/rules/blob/aeaf93bc058408e260192d0941a688963449d6be/src/rules/simple-user-whitelist-for-app.js).

* Pour restreindre l’accès des utilisateurs aux applications, vous pouvez configurer une nouvelle connexion et n’accorder l’accès qu’à une application précise.

* Allez à [Dashboard > Authentification > Sociale](%24%7Bmanage_url%7D/#/connections/social) et cliquez sur une connexion.
* Dans l’onglet **Applications**, activez/désactivez les applications.

* Pour désactiver l’accès des utilisateurs à vos applications, vous pouvez [Bloquer et débloquer des utilisateurs](/users/block-and-unblock-users) dans Dashboard.
