Le profil utilisateur normalisé est un moyen spécifique à Auth0 de normaliser et de stocker les informations relatives à l’utilisateur. Étant donné que chaque fournisseur d’identités (IdP) fournit un ensemble différent d’informations sur un utilisateur, Auth0 normalise les propriétés communes du profil en une représentation de l’utilisateur qui ne dépend pas du protocole lors du stockage des demandes liées à l’utilisateur. Par exemple, `family_name` dans le profil utilisateur normalisé contient des détails qui peuvent être renvoyés à un IdP sous la forme de `surname` ou `last_name`.

Ainsi, les demandes Auth0 comprises dans le profil utilisateur normalisé diffèrent de l’ensemble standard de demandes qui peuvent être retournées dans les jetons d’ID provenant du [point de terminaison `oauth/token` de l’Authentication API](https://auth0.com/docs/api/authentication#get-token) ou dans la réponse du point de terminaison [`/userinfo`](https://auth0.com/docs/api/authentication#user-profile) (ces derniers suivant les exigences détaillées dans la spécification OIDC).

Pour en savoir plus sur les attributs inclus dans le profil utilisateur normalisé et comprendre comment vous pouvez les utiliser, lisez [Structure du profil utilisateur](/users/user-profiles/user-profile-structure).
