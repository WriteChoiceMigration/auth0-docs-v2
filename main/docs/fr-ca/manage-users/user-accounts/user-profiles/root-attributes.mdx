Le profil utilisateur normalisé d’Auth0 contient un certain nombre d’attributs racine (attributs stockés au premier niveau, ou niveau racine, de l’objet), dont certains peuvent être mis à jour. Pour en savoir plus sur la structure du profil utilisateur et ses attributs, consultez [Structure du profil utilisateur](/users/user-profiles/user-profile-structure)

Les méthodes de mise à jour des attributs racine varient en fonction du type de connexion.

## Auth0 en tant que fournisseur d’identité

Lorsque Auth0 est le fournisseur d’identité (IdP), les abonnés peuvent :

* Définir les attributs racine [lors de l’inscription de l’utilisateur](/api/management/guides/users/set-root-attributes-user-signup) (via Management API ou une inscription publique) ou [lors de l’importation](/api/management/guides/users/set-root-attributes-user-import)
* [Mettre à jour les attributs racine individuellement](/api/management/guides/users/update-root-attributes-users) via Management API.

Auth0 est l’IdP pour les types de connexion suivants :

* [connexions par bases de données ordinaires](/connections/database)
* [connexions par bases de données personnalisées](/connections/database/custom-db) avec mode d’importation,
* [connexions sans mot de passe](/connections/passwordless)

## Fournisseurs d’identité en amont

Lorsqu’un IdP en amont (comme Google ou Facebook) est utilisé, deux options s’offrent aux abonnés :

* L’IdP en amont définit les attributs racine lorsque les utilisateurs sont créés pour la première fois, puis les met automatiquement à jour lors de chaque connexion ultérieure. Il s’agit du comportement par défaut.
* L’IdP en amont définit les attributs racine uniquement lors de la création de l’utilisateur et ne les met pas à jour lors des connexions ultérieures, ce qui permet aux abonnés de [mettre à jour les attributs racine individuellement](/api/management/guides/users/update-root-attributes-users) via Management API. Pour activer cette option, vous devez [configurer votre synchronisation de connexion avec Auth0](/api/management/guides/connections/configure-connection-sync).

Les fournisseurs d’identité en amont gèrent les types de connexion suivants :

* [connexions par médias sociaux](/connections#social)
* [connexions d’entreprise](/connections#enterprise)
* [connexions d’identité légale](/connections#legal-identities)
