テナント管理者であれば、同僚をテナントに追加し、特定のロールを割り当てることで、必要なAuth0 Dashboard機能へのアクセスを許可することができます。



新しいテナントメンバーを追加する際、ロールを割り当てることで、運用環境のアプリケーションや機密情報を危険にさらすことなくアクセスを許可することができます。ユーザーには複数のロールを割り当てることができますが、その場合、アクセス権は統合されます。保護を強化するために、ユーザーは[多要素認証](/manage-dashboard-access/add-change-remove-mfa)（MFA）を有効にすることができます。各ロールでアクセス可能なダッシュボード機能の詳細については、「[ロール別のDashboardへのアクセス](/manage-dashboard-access/feature-access-by-role)」を参照してください。





1. [［Dashboard］>［Settings（設定）］>［Tenant Members（テナントメンバー）］](%24%7Bmanage_url%7D/#/tenant/admins)に移動します。
2. **［Add Member（メンバーの追加）］**をクリックします。
3. **［Add New Tenant Member（新しいテナントメンバーの追加）］**画面で、ユーザーのメールアドレスを入力し、割り当てるロールを選択します。
4. **［Invite（招待）］**をクリックします。Auth0はユーザーに、クリックすると招待を受け入れることができるリンク付きのメールを送信します。ユーザーが招待を受け入れる前は、招待リストに保留中の招待として表示されます。`**...**`をクリックすると、**［Remove invitation（招待の削除）］**または**［Copy invitation link（招待リンクのコピー）］**が表示されます。



招待されたユーザーがリンクをクリックして招待を受け入れると、そのユーザーは招待されたメールアドレスでログインするよう求められます。ユーザーはその後、ソーシャルプロバイダーでログインするか、メールアドレスとパスワードでログインするかを選択できます。メールアドレスとパスワードでログインする場合は、すでにAuth0のアカウントが作成されている必要があります。まだの場合は、まずサインアップしてから、招待を再度受け入れる必要があります。
