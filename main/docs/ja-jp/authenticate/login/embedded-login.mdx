Auth0の埋め込みログインを使用すると、Auth0サーバーに資格情報を送信し、ユーザーをアプリケーションに直接ログインさせることができます。埋め込みログインの使用は推奨しません。詳細については、「[中央管理のユニバーサルログインと埋め込みログイン](/login/universal-vs-embedded-login)」を参照してください。

埋め込みログインを使用すると決めた場合には、[アプリケーションにクロスオリジンのリソース共有（CORS）を構成](/applications/set-up-cors)しなければなりません。また、[カスタムドメインを構成](/custom-domains)する必要があります。その上で、アプリケーション内に[Lock SDK](/libraries/lock)または[Auth0.js SDK](/libraries/auth0js)を実装するか、[Auth0 Authentication API](https://auth0.com/docs/api/authentication)を直接呼び出します。

Auth0は複数のアプリケーションタイプのパスワードレス接続で埋め込みログインに対応しています。

* [ネイティブアプリケーションでの埋め込み型パスワードレスログイン](/native)
* [通常のWebアプリケーションでの埋め込み型パスワードレスログイン](/webapps)
* [シングルページアプリケーションでの埋め込み型パスワードレスログイン](/spa)
