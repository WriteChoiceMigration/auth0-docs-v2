Auth0 DashboardまたはManagement APIを使用して、ユーザーに[権限](/rbac)を割り当てることができます。割り当てられた権限は、API Authorization Core機能セットで使用できます。



## 前提条件

* 
* Dashboardで[APIをセットアップ](/get-started/set-up-apis)します。
* 権限は事前に定義された値から選択されます。権限のリストが空白の場合は、APIに[権限を追加](/apis/add-api-permissions)してください。

## Dashboard

ユーザーに権限を割り当てる方法は2つあります。**［Users（ユーザー）］**リストからユーザーを選んで権限を割り当てるか、各ユーザーの**［User Details（ユーザー詳細）］**（ユーザープロファイル）ページに移動して、**［Permissions（権限）］**タブで権限を選んで割り当てることができます。

### ユーザーリストで権限を割り当てる

1. [［Dashboard］>［User Management（ユーザー管理）］>［Users（ユーザー）］](%24%7Bmanage_url%7D/#/users)に移動します。
2. 変更したいユーザーの横にある`**［...］**`をクリックし、**［Assign Permissions（権限の割り当て）］**を選択します。
3. 権限の割り当てに使用するAPIを選択し、ユーザーに割り当てる権限を選択してから、**［Add Permissions（権限の追加）］**をクリックします。

### ユーザープロファイルで権限を割り当てる

個人のプロファイルページからユーザーに権限を割り当てることもできます。

1. [［Dashboard］>［User Management（ユーザー管理）］>［Users（ユーザー）］](%24%7Bmanage_url%7D/#/users)に移動し、ユーザー名をクリックして表示します。
2. **［Permissions（権限）］**タブをクリックし、**［Assign Permissions（権限の割り当て）］**をクリックします。
3. 権限の割り当てに使用するAPIを選択し、ユーザーに割り当てる権限を選択してから、**［Add Permissions（権限の追加）］**をクリックします。

## Mangement API

`POST`呼び出しを[ユーザー権限割り当てエンドポイント](https://auth0.com/docs/api/management/v2#!/Users/post_permissions)に対して行います。`USER_ID`、`MGMT_API_ACCESS_TOKEN`、`API_IDENTIFIER`、`PERMISSION_NAME`のプレースホルダーをそれぞれユーザーID、Management APIのアクセストークン、API識別子、権限名に置き換えます。
