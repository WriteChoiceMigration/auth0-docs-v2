Auth0のコア認可機能は、APIのRole-Based Access Control（RBAC） に使用することができます。

Role-Based Access Control（RBAC）を正しく動作させるには、ダッシュボードまたはManagement APIを使用してAPIを有効にしなければなりません。詳細は、「[APIのRole-Based Access Controlを有効にする](/apis/enable-role-based-access-control-for-apis)」をお読みください。

認可コア機能は、認可拡張機能とは異なります。これら2つの違いについては、[「Authorization Core（認可コア）と認可拡張機能」](/rbac/authorization-core-vs-authorization-extension)をお読みください。

コア機能をより効果的に使用するには、以下を行います。

1. [APIをAuth0に登録する](/get-started/set-up-apis)
2. [APIにアクセス許可を定義する](/apis/add-api-permissions)
3. [ロールを作成する](/rbac/roles/create-roles)
4. [ユーザーにロールを割り当てる](/rbac/rbac-users/assign-roles-to-users)
5. [ユーザーにアクセス許可を割り当てる](/rbac/rbac-users/assign-permissions-to-users)（必要な場合）
