ユーザーがパスワードを正常にリセットした後にリダイレクトされるURIは、以下のすべての条件に影響を受けます。

1. テナントがユニバーサルログインを使用するか、クラシックログインを使用するか、および
2. Management APIを直接使用してパスワード変更チケットを生成するかどうか、または
3. Authentication API、ログインページ、または他のソースからパスワード変更のメールを送信するか

次のフローチャートは、テナントのリダイレクトURIを構成に応じて設定するためのガイドです。
