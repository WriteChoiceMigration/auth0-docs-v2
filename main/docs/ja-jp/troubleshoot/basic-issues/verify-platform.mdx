テクノロジースタックが最新バージョンであることを確認します。プログラミング言語またはライブラリの古いバージョンを使用している場合は、最新バージョンに更新することを検討してください。

* Check the [Auth0 support matrix](/docs/troubleshoot/customer-support/product-support-matrix) to see if the language or library versions you're using is supported.
* 1 つの環境（開発など）で問題が発生し、別の環境（本番など）では問題が発生しない場合は、2つの環境間でテクノロジースタックのバージョンを比較します。
* そうでない場合は、テクノロジースタックとブラウザを最新バージョンにアップグレードし、サポートリクエストを作成する前に再度テストします。

## 問題の範囲を確認する

Testing on different browsers, platforms, locations, and users can help narrow down the source of a problem when it occurs. Perform the following tests and review the [logs in your Auth0 dashboard](https://manage.auth0.com/#/logs) after each test for more information.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

See [the Monitoring page](/docs/deploy-monitor/monitor) for more details on each of the log events that can help you troubleshoot issues.

</Callout>

### さまざまなブラウザでテストする

* 問題はすべてのブラウザで発生しますか、それとも一部のブラウザでのみ発生しますか。
* 新しいシークレットモードまたはプライベートブラウザセッション（以前の状態なし）でテストします。
* ブラウザの最新バージョンにアップグレードして再度テストします。
* ブラウザプラグインをオフにして再テストし、プラグインが問題の原因になっているかどうかを確認します。

### 異なるユーザーでテストする

* 問題はすべてのユーザーで起きていますか、それとも一部のユーザーで起きていますか。
* 一部のユーザーで発生する場合、パターンはありますか。場所、接続、役割などはどうですか。

### 異なる接続でテストします（可能な場合）

* 問題はすべての接続で発生しますか、それとも1つの接続で発生しますか。

### 異なるクライアントアプリケーションでテストします（複数のクライアントがある場合）

* 問題はすべてのクライアントで発生しますか、それとも1つのクライアントで発生しますか。

### 異なる環境でテストする

* アプリケーションの開発環境、ステージング環境、および本番環境で試します。
* 異なるAuth0テナントで試して、違いがあるかどうかを確認します。

## 問題が一貫しているか断続的かを確認する

* 問題が一貫している場合は、問題を確実に再現するための手順を書き留めます。
* 問題が断続的に発生する場合は、原因となる可能性のあるパターンを書き留めてください。
* これは新しいセットアップの問題ですか、それとも以前は機能していたが今は機能しなくなったものですか。

  + 問題が突然発生した場合、どのような変更が行われましたか。
  + 変更を元に戻すと問題は解決しますか。

## 1人または少数のユーザーのみに影響する問題

* [Check the user's profile](/docs/troubleshoot/authentication-issues/check-user-profiles), browser, or device for any issues.
* 影響を受けるユーザーのすべてのブラウザで問題が発生するかどうか（データの問題を示します）、または特定の種類のブラウザでのみ発生するかどうか（ブラウザ固有の問題を示します）を確認します。
* ブラウザでJavaScriptとCookieが有効になっているかどうかを確認します。
* Caps Lockキーが無効になっていることを確認します。
* ユーザーがモバイルデバイスを使用している場合は、認証や承認に影響を与える可能性のあるソフトウェアがないかどうかを確認します（必要なソフトウェアが実行されていないなど）。
* Check to see if the user can access some of the application's key URLs, such as the <Tooltip tip="Identity Provider (IdP): Service that stores and manages digital identities." cta="View Glossary" href="/docs/glossary?term=identity+provider">identity provider</Tooltip>'s <Tooltip tip="Single Sign-On (SSO): Service that, after a user logs into one applicaton, automatically logs that user in to other applications." cta="View Glossary" href="/docs/glossary?term=Single+Sign-on">Single Sign-on</Tooltip> (SSO) URL (indicating a network connectivity issue).

## 稼働開始後に発生する問題（以前は機能していたが、その後機能しなくなった）

* アプリケーションまたは呼び出すAPIに最近変更があったかどうかを確認します。
* ネットワークに最近変更があったかどうかを確認します（ロードバランサー、ファイアウォール、プロキシ構成の変更など）。
* インフラストラクチャに最近変更があったかどうかを確認します（LDAPまたはDBサーバーの資格情報など）。
* アプリケーション、インフラストラクチャ、またはテクノロジースタックにパッチまたは更新があったかどうかを確認します。
* すべてのサーバーがNTPを実行し、正確な時刻同期が行われていることを確認します。
* 証明書の有効期限が切れていないかどうかを確認します。
* リモートIDプロバイダーの所有者に、変更があったかどうかを確認します。

  + リモートIDプロバイダーへのネットワーク接続を確認します。
* Check notifications in <Tooltip tip="Auth0 Dashboard: Auth0's main product to configure your services." cta="View Glossary" href="/docs/glossary?term=Auth0+dashboard">Auth0 dashboard</Tooltip>/migrations - any changes you overlooked?
* Auth0変更ログを確認します。問題に関連する最近の構成変更はありますか。
* テクノロジースタックのコンポーネントが最近更新またはパッチ適用されたかどうかを確認します。
