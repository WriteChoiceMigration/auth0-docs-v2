Auth0サブスクリプションプランによって、ユーザーの制限と利用できる機能が決まります。

## クォータ使用率

You can view subscription utilization information from Auth0 Support Center by visiting [Reports > Quota Utilization](https://support.auth0.com/reports/quota). The subscription utilization information is available with the following details:

* ユーザー消費率

  + アクティブユーザー
* 機能使用率

  + エンタープライズ接続
  + マシンツーマシン認証
  + 多要素認証

<Frame>![Reports showing subscription feature usage](/images/cdy7uua7fh8z/7g0PDAKl5D8YEf4vC5FjSI/188182aabbeb7e0af77b1bc9d292cfad/Screenshot_2023-02-07_at_5.35.40_PM.jpeg)</Frame>

詳細なサマリーカードごとに、現行月の使用率が利用可能なサブスクリプションの制限と対比する形で表示されます。また、（カードをクリックして）さらに掘り下げていき、Auth0プライベートインスタンス（該当する場合）とテナントによる使用状況の内訳などの詳細を最大12か月前までさかのぼって表示することもできます。

必要に応じて、CSVをダウンロードしてこのデータをエクスポートするオプションもあります。

<Frame>![](/images/cdy7uua7fh8z/7bBqDuycQWsnUdd9hlXN10/040a04d458bff06e8392c1432e3308ce/Screenshot_2023-02-07_at_6.07.01_PM.jpeg)</Frame>

### アクティブユーザー

これによって、特定のカレンダー月のアクティブなイベント（ログインまたはサインアップ）で、一意のユーザーのカウントを取得することができます。掘り下げていき、これらのアクティブユーザーの内訳をテナントとプライベートのAuth0インスタンス（該当する場合）別に最大12か月前までさかのぼって入手することもできます。

### エンタープライズ接続

An [Enterprise connection](/docs/authenticate/enterprise-connections) is considered active if (during the current month) it has both:

* ユーザーアクティビティ（ログイン、サインアップ、トークン交換など）があった。
* エンタープライズ接続がアプリケーションに対して一度も有効になったことがない、または有効であったものの、現行月でユーザーアクティビティが一切見られなかった場合、アクティブであるとはみなされません。

このレポートを使って、プライベートAuthのインスタンス（該当する場合）ごとのアクティブな接続の内訳を含む特定のカレンダー月のアクティブなエンタープライズ接続のカウントだけでなく、アクティブなエンタープライズ接続とテナント別に示したアクティブユーザーのカウントを取得できます。

マシンツーマシン認証

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

The Okta Workforce Enterprise connection is not counted toward your active エンタープライズ接続 limit.

</Callout>

### このレポートでは、[クライアントの資格情報フロー](/login/flows/client-credentials-flow)（別名：API認証呼び出し）に対して発行されたアクセストークンのカウントを確認できます。このレポートはカレンダー月ごとに発行され、Auth0 Management APIに対して発行されたトークンは除外されます。

This report provides a count of <Tooltip tip="Access Token: Authorization credential, in the form of an opaque string or JWT, used to access an API." cta="View Glossary" href="/docs/glossary?term=access+tokens">access tokens</Tooltip> issued for the [Client Credentials Flow](/docs/get-started/authentication-and-authorization-flow/client-credentials-flow), also referred to as API Authentication Calls, per calendar month excluding any tokens issued for Auth0 <Tooltip tip="Management API: A product to allow customers to perform administrative tasks." cta="View Glossary" href="/docs/glossary?term=Management+API">Management API</Tooltip>.

Auth0パブリッククラウドのデプロイメントオプションのテナントには、過去7日間における認証呼び出しの内訳もテナントアプリケーション別に表示されます。

多要素認証

### 多要素認証（MFA）の使用状況を示したレポートは2つあり、カレンダー月ごとのMFAイベント（登録またはチャレンジ）の数を示します。

There are two reports for 多要素認証（MFA）の使用状況を示したレポートは2つあり、カレンダー月ごとのMFAイベント（登録またはチャレンジ）の数を示します。 (<Tooltip tip="Multi-factor authentication (MFA): User authentication process that uses a factor in addition to username and password such as a code via SMS." cta="View Glossary" href="/docs/glossary?term=MFA">MFA</Tooltip>) usage which present a count of MFA events (enrolments or challenges) by calendar month.

これらの両方のレポートタイプには、MFAカウントの内訳がAuth0プライベートインスタンス*（該当する場合）とテナント別に表示されます。

Both these report types provide a breakdown of MFA counts by Auth0 Private Instance\* (if applicable) and Tenant.

\*Auth0 Private Instance MFA usage is only available since 1st January 2023.

## アクティブユーザー：月間アクティブユーザー数、ユーザータイプ（ソーシャル、パスワード、パスワードレス、エンタープライズ）、および総アクティブユーザー数

You can view total and active user counts by calendar month from Auth0 Support Center for a given tenant by visiting [Reports > Usage](https://support.auth0.com/reports/usage), and selecting a Subscription (and a Tenant for self-service subscriptions). This would provide the following breakdown:

* Active Users: Count of active users by month and user type (Social, Password, <Tooltip tip="Passwordless: Form of authentication that does not rely on a password as the first factor." cta="View Glossary" href="/docs/glossary?term=Passwordless">Passwordless</Tooltip>, Enterprise) along with a Total Active count.
* アクティブユーザー by Tenant: Count of active users by month, tenant, and user type (Social, Password, Passwordless, Enterprise) along with a Total Active count.
* Total Users by Tenant

<Frame>![Support Center -> Reports -> Usage](/images/cdy7uua7fh8z/2RmWp1ZIWwV2JCPJcftll5/acd9c3c71b8dbd9cca12a44588e015af/Screen_Shot_2022-10-27_at_1.21.35_PM.png)</Frame>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

Counts of Total Active or Total Users can be higher than the sum of users by each individual type for situations where user type is not determined. This can happen, but is not limited to, scenarios where account linking is turned on.

</Callout>
