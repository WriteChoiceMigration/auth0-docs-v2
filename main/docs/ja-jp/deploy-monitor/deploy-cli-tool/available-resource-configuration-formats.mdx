Auth0リソースの状態は、2つの使用可能な異なる構成ファイル形式で表現されます。YAMLとディレクトリ（JSON）。Deploy CLIの`export`コマンドを使用すると、一方と他方の選択が提示されます。

## YAML

YAML形式は、アクションやメールテンプレートなどのリソースの補助コードファイルとともに、主にフラットなtenant.yamlファイルとして表現されます。単一のファイルにより、バージョン管理での時間経過に伴う変更の追跡がわかりやすくなります。さらに、単一のファイルは、直ちに明らかでない可能性があるディレクトリやファイル名とのあいまいさを若干排除します。

## ディレクトリ（JSON）

The directory format separates resource types into separate directories, with each single resource living inside a dedicated JSON file. This format allows for easier conceptual separation between each type of resource as well as the individual resources themselves. Also, the Deploy CLI closely mirrors the data shapes defined in the [Auth0 Management API](https://auth0.com/docs/api/management/v2), so referencing the JSON examples in the docs may be useful when using this format.

## 選択の方法

使用する形式を選択する決定は、主に好みに基づいて行うべきです。両方の形式は、上記のわずかに異なる強みと弱みがありますが、同じタスクを実現する有効な解決策です。自身のコンテキストでそれぞれを確実に評価してください。重要な点は、この選択は永続的なものではなく、`import`コマンドを使用して一方から他方に自由に切り替えることができます。
