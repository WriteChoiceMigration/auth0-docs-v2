The Private Cloud deployment option comes with two additional add-on features: Geo-failover and PCI compliance. If you have any questions about these features, please [contact Auth0 Sales](https://auth0.com/get-started?place=documentation%20post&type=link&text=contact%20auth0%20sales).

## Private Cloud Geo-Failover

ジオフェイルオーバーはアクティブ･パッシブフェイルオーバー構成であり、大規模なリージョン停止が発生した場合に使用するセカンダリフェイルオーバーリージョンをお客様が指定できます。一般に、同じクラウドプロバイダー内の100ミリ秒以内の任意のリージョンをフェイルオーバーリージョンとして選択できます。場合によっては、クラウドプロバイダーが特定のフェイルオーバーリージョンを優先します。Auth0オンボーディングチームがお客様と協力して、ユースケースに最適なリージョンを選択します。

さらに、データストアの1つに第3の「タイブレーク」リージョンが必要です。この第3のリージョンはライブトラフィックを受信することはありませんが、フェイルオーバーが発生した場合に新しいリーダーを選出するために必要です。Auth0は、このリージョンに最適なオプションを選択するお手伝いもします。

リージョン障害が検出されると、Auth0がお客様に代わってジオフェイルオーバーをトリガーします。トリガーされると、プロセス全体が自動化されます。リージョンで問題が発生しなくなったことが確認されると、Auth0はプライマリリージョンにフェイルバックします。

## PCI Compliance

PCIコンプライアンスは、必要なお客様にアドオンとしてご利用いただけます。
