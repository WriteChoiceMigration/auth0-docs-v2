複数のエンタープライズアプリケーションでユーザーのSSOログインを管理するために、​[](/extensions/single-sign-on-dashboard-extension)SSOダッシュボード拡張機能を使用します。

詳細については、[シングルサインオンダッシュボード拡張機能](/extensions/single-sign-on-dashboard-extension)を参照するか、または[GitHubでこの拡張機能を確認してください](https://github.com/auth0-extensions/auth0-sso-dashboard-extension)。

## 前提条件

* Auth0で[SSOダッシュボードアプリケーションを作成します](/extensions/single-sign-on-dashboard-extension/create-sso-dashboard-application)。
* [SSOダッシュボード拡張機能をインストールします](/extensions/single-sign-on-dashboard-extension/install-sso-dashboard-extension)。
* [SSOダッシュボードにアプリケーションを追加します](/extensions/single-sign-on-dashboard-extension/add-applications-to-the-sso-dashboard)。

## アプリケーションを更新

1. [［Auth0 Dashboard］>［Extensions（拡張機能）］](%24%7Bmanage_url%7D/#/extensions)に移動して、**［Installed Extensions（インストール済みの拡張機能）］**ビューを選択します。
2. **［SSO Dashboard（SSOダッシュボード）］**を選択します。管理者である場合、`https://${account.tenant}.<REGION>8.webtask.io/auth0-sso-dashboard/admins/login`を使用して、SSOダッシュボードにログインすることもできます。
3. テナント名の下にあるドロップダウンメニューから**［Settings（設定）］**を選択します。

* アプリケーションが有効になっている場合にユーザーが確認できるか否かを指定するには、**［Publish（公開）］**または**［Unpublish（非公開）］**を選択します。
* アプリケーション設定を更新するには、そのギアアイコンを選択します。
* アプリケーションを削除するには、その**［X］**を選択します。削除を確認するポップアップが表示されます。
