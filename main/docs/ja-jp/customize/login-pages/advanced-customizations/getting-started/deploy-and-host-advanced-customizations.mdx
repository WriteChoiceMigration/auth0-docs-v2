以下のガイドはカスタマイズしたユニバーサルログイン画面を運用環境にデプロイして、継続的インテグレーションと継続的な配信パイプラインを作成するのに役立ちます。

## プロジェクトをセットアップする

アップロードする画面をすべて準備して、ACULプロジェクトに以下のファイル構造を使用します。

### 新しいVite構成を作成する

以下を`vite.config.ts`という名前の新しいファイルに追加します。



### ビルドスクリプトをセットアップする

`scripts/build.ts`を作成してViteのビルドを実行します。



### AWSインフラストラクチャを構築する

S3バケットとCloudFrontディストリビューションを作成します。



### Auth0にCORSを構成する



### デプロイメントスクリプトを作成する

AWSのデプロイメントに`scripts/deploy.ts`を作成します。



### GitHub Actionsのワークフローをセットアップする

`.github/workflows/deploy.yml`を作成します。



最後に、以下のシークレットをGitHubリポジトリに追加します。

* `AWS_ACCESS_KEY_ID`
* `AWS_SECRET_ACCESS_KEY`
* `CLOUDFRONT_ID`
* `CLOUDFRONT_DOMAIN`
* `AUTH0_DEPLOY_TOKEN`
